# Ozon生活家居选品分级标准

## 1. 使用边界

本文用于对 `docs/Ozon/选品数据.xlsx` 中已经初筛出的生活家居候选品做二次筛选。当前阶段不建议直接批量上架，而是先按 A/B/C 三类标记，区分“优先小批量测试”“继续补充调研”“暂不操作”。

判断重点不是单一销量，而是综合判断：

- 是否有稳定需求
- 是否有真实利润空间
- 是否能稳定履约
- 是否存在合规、侵权、退货、破损等风险
- 是否具备跨境卖家的可操作优势

## 1.1 初筛核心判断逻辑

AI 初筛的目标不是直接决定上架，而是判断“这个 SKU 是否值得进入下一轮人工核价和供应链调研”。因此不采用简单平均打分，而采用四层判断：

1. 真实需求：优先看销售额规模、销量周转、加购率、订单转化率、签收率。增长率只能作为辅助证据，不能单独决定 A 类。
2. 利润空间：优先看预估毛利率，但只作为初判。预估毛利率不能替代真实采购价、物流费、平台费用、广告成本和退货损耗后的净利润。
3. 物流履约友好度：优先看长宽高、重量、客单价与签收率。低客单大体积、低毛利中高重量、易碎易退品需要降级到 B 类或 C 类。
4. 一票否决风险：品牌侵权、合规认证、高风险属性、明显本土壁垒、强广告依赖且转化弱等，不应被销量或毛利掩盖。

决策公式：

```text
优先测试候选 = 真实需求成立 × 利润空间够 × 物流履约友好 × 无明显一票否决风险
```

广告依赖、转化质量、签收质量不是独立的最终目标，而是用来解释真实需求和利润是否会被广告、退货、履约成本吞掉。

## 2. 三类选品标准

### A类：优先小批量测试

适合进入小批量上架测试池，但不等于大批量压货。

判定标准：

- 销量或销售额在最近周期内稳定增长，不是短期异常爆发。
- 所属细分类目供给不算拥挤，且销售额表现较好。
- 预估毛利率较高，补齐采购成本、物流费、平台费用、广告费用、退货损耗后仍有净利润空间。
- 商品体积、重量、包装难度可控，跨境运输成本不会吞掉利润。
- 不明显涉及带电、液体、食品接触、儿童安全、刀具、磁性、化妆品、医疗宣称等高风险因素。
- 商品信息中显示跨境允许销售，且没有明显品牌、外观、图片、专利侵权风险。
- 竞品转化、加购、签收表现相对健康，广告占比不过度异常。
- 供应链稳定，能拿到明确采购价、起订量、发货时效和补货周期。
- 竞品评论或详情页存在可优化点，例如尺寸说明不清、包装差、图片不准、组合规格不合理。

AI 初筛时，A候选必须同时满足：需求信号强、毛利空间强或中高、物流风险不高、没有明显一票否决风险。A候选仍然需要补采购价和物流费后才能变成最终 A 类。

建议动作：

- 小批量上架测试。
- 控制首批库存和广告预算。
- 重点验证点击率、转化率、签收率、退货/未取件、结算后利润。

### B类：继续补充调研

数据表现有机会，但关键风险或成本没有验证清楚，暂时不应直接上架。

判定标准：

- 销量、销售额或潜力指数较好，但真实采购成本、物流费、平台费用没有算清楚。
- 预估毛利率看起来可以，但广告占比、签收率、转化率存在疑点。
- 商品尺寸、重量、易碎性、包装方式存在不确定性。
- 可能涉及认证、材质说明、俄文说明、食品接触、儿童使用场景等风险，需要进一步确认。
- 竞品是本土卖家为主，跨境卖家是否有时效或成本劣势还不清楚。
- 商品同质化较强，但尚未明确差异化角度。
- 供应商链接、采购价、MOQ、发货时效、质检稳定性等信息缺失。

AI 初筛时，B待补是默认的谨慎池：需求可能成立，但利润、物流、合规、侵权、广告依赖或供应链还没有足够证据。B 类不是差品，而是“不能跳过人工核验”的品。

建议动作：

- 暂不上架。
- 补充采购价、物流报价、平台费用、合规风险、差评痛点、供应链稳定性。
- 补齐后重新判断是否升为 A 类或降为 C 类。

### C类：暂不操作或保留观察

当前不建议投入上架资源。

判定标准：

- 销量好但利润薄，扣除物流、平台费用、广告、退货后净利率不可接受。
- 客单价低但体积、重量、包装成本偏高。
- 同质化严重，只能依靠低价竞争，跨境卖家没有明显优势。
- 易碎、易退、尺寸敏感、色差敏感，售后风险高。
- 涉及高风险合规、认证、侵权、品牌限制，短期无法排除。
- 本土卖家明显占优，且配送时效、价格、评价数构成明显壁垒。
- 广告依赖过高，或转化、签收表现弱，说明真实需求质量不足。
- 供应链不稳定，无法保证质量、包装、补货周期或售后处理。

AI 初筛时，C观察主要用于明显不值得当前投入人工补数的 SKU，例如低客单低毛利、物流体积风险过高、品牌/合规风险过高、转化与签收共同偏弱的产品。

建议动作：

- 暂不操作。
- 仅保留观察，除非后续拿到明显供应链优势、差异化方案或风险排除证据。

## 4. 最新打标字段分组

打标只分两组，避免把选品工作变成大量人工填表。

### 4.1 AI自动初筛字段

AI 只基于现有 `选品数据.xlsx` 自动生成以下字段。用户不需要手填这些字段。

| 字段 | 建议取值 | 说明 |
| --- | --- | --- |
| AI初筛分类 | A候选 / B待补 / C观察 | 第一轮分流，不等于最终上架结论 |
| AI下一步动作 | 小批量测试前补成本 / 补关键调研 / 暂不操作 / 先确认侵权风险 / 先核物流费和包装风险 | 告诉用户下一步具体做什么 |
| AI判断依据摘要 | 需求、利润、物流、风险摘要 | 用四层逻辑解释为什么这样分 |
| 真实需求判断 | 强 / 中 / 弱 | 由销量周转、销售额规模、加购、转化、签收共同判断 |
| 利润空间初判 | 强 / 中 / 边界 / 弱 / 不确定 | 只基于预估毛利率做初筛，不替代真实净利 |
| 物流履约判断 | 低 / 中 / 中偏高 / 高 | 由长宽高、重量、客单价、签收表现共同判断 |
| 一票否决风险初判 | 无明显风险 / 品牌侵权风险 / 合规风险 / 物流风险 / 签收风险 / 流量风险等 | 标出不能被销量和毛利掩盖的风险 |
| 是否需要人工补充 | 是 / 否 | C观察默认不补，A候选和关键 B待补才补 |
| 人工补充优先级 | P1 / P2 / P3 / 不补 | 控制人工补数顺序 |

保留广告依赖、转化质量、签收质量、履约方式参考、价格竞争风险等辅助字段，但它们只用于解释四层判断，不作为独立的最终决策中心。

### 4.2 用户真实补充字段

用户只对 `A候选` 和 `P1级 B待补` 补以下字段。`C观察` 默认不补，除非后续重新评估。

| 字段 | 说明 |
| --- | --- |
| 采购价 RMB | 计算真实净利的基础 |
| 预估物流费 RMB | 判断跨境物流是否吞掉利润 |
| 供应商链接 | 用于核价、补货、比价 |
| 侵权风险人工确认 | 品牌、外观、图片、专利等风险 |
| 合规风险人工确认 | 材质、认证、用途宣称、食品接触、儿童场景等风险 |
| 竞品差评痛点 | 判断是否有可优化切入点 |

## 5. 最新执行方式

1. AI 先完成初筛，不让用户给所有 SKU 手工打标。
2. 用户只补 A候选和 P1级 B待补的真实商业信息。
3. 补完后再做最终 A/B/C 分类。
4. 最终分类必须结合采购价、物流费、侵权风险、合规风险和竞品差评痛点，不能只依赖 AI 初筛。

执行原则：

```text
AI负责省时间：先排除低优先级 SKU
用户负责补真相：只补会影响最终决策的真实成本和风险信息
最终决策负责止损：只有真实需求、利润、物流、风险都过关，才进入小批量测试
```

# 数据库说明文档

## 1. 数据库定位

电商工作台数据库用于承接采集任务、商品经营快照、商品内容资产和原始采集审计数据。数据库不绑定单一数据源，后续可以接入萌拉、Ozon、1688、广告数据或其它电商运营来源。

目标数据库名称：

```text
db/ecommerce-workbench.sqlite
```

旧数据库名称：

```text
db/menglar-mvp.sqlite
```

旧名称仅作为迁移来源保留，不再作为后续默认数据库名称。

## 2. 核心数据分层

当前重构后的数据库按数据性质分为三类业务数据，以及一类技术审计数据。

| 数据层 | 表名 | 作用 |
| --- | --- | --- |
| 采集任务数据 | `source_jobs` | 记录一次采集任务什么时候执行、从哪里采集、执行状态和结果摘要 |
| 商品经营数据 | `product_business_snapshots` | 记录某个平台商品在某一次采集任务中的销量、销售额、曝光、点击、转化、物流、尺寸重量等经营表现 |
| 商品内容数据 | `product_content_assets` | 记录商品标题、描述、属性、标签、图片 URL、图片下载结果等内容资产 |
| 原始审计数据 | `products_raw` | 保存原始采集载荷，用于排错、回放和重新解析 |

## 3. 表关系

```text
source_jobs
  1 -> N products_raw

source_jobs
  1 -> N product_business_snapshots

products_raw
  1 -> 0..1 product_business_snapshots

product_business_snapshots
  通过 platform + platform_product_id
  关联同一商品的多次经营快照

product_content_assets
  通过 platform + platform_product_id
  关联商品内容资料
```

## 4. 商品身份规则

当前阶段不单独建设 `product_master` 商品主档。商品身份由业务表中的稳定来源字段表达：

```text
platform + platform_product_id
```

如果数据源暂时没有稳定平台商品 ID，才允许退化使用：

```text
platform + normalized_product_url
```

不建议使用品牌、标题、规格做主去重规则，因为这些字段容易变化，也容易把相似商品误判为同一个商品。

## 5. 迁移原则

本次重构属于开发期破坏性结构调整，但仍保留可追溯迁移路径。

1. 数据库默认文件名从 `db/menglar-mvp.sqlite` 改为 `db/ecommerce-workbench.sqlite`。
2. `products_normalized` 停止作为业务表使用。
3. 新建 `product_business_snapshots`，承接原 `products_normalized` 的经营快照数据。
4. 新建 `product_content_assets`，承接商品详情、描述、属性、标签、图片等内容资产。
5. `source_jobs` 暂时不调整字段结构。
6. `products_raw` 暂时不调整字段结构，继续作为原始审计表。
7. 旧表 `products_normalized` 在迁移完成后可保留一段时间用于对账，待采集、展示和筛选链路稳定后再删除。

## 6. 页面与功能对应

| 功能 | 主要读取表 | 说明 |
| --- | --- | --- |
| 采集任务页 | `source_jobs` | 展示采集任务状态、数量、错误类型和执行时间 |
| 结果展示页 | `source_jobs` + `product_business_snapshots` | 按批次展示原始经营快照，用于初筛 |
| 商品筛选页 | `product_business_snapshots` + `product_content_assets` | 展示初筛商品的二次处理状态、内容补充和图片处理情况 |
| 数据排错 | `products_raw` | 对照原始载荷和解析结果，定位字段变化或解析失败 |

## 7. 后续扩展

后续如果进入上货准备和长期商品管理阶段，再新增以下表，不建议提前塞进当前三类数据表：

| 表名 | 作用 |
| --- | --- |
| `product_selection_pool` | 人工筛选池、候选/排除/待处理状态 |
| `product_publish_drafts` | 上货草稿、标题、类目、价格、库存、物流等发布字段 |
| `product_image_downloads` | 更细粒度的图片下载任务和文件状态 |
| `source_pages` | 固定采集入口配置，适用于稳定维护多个采集入口后再拆 |

# 萌拉采集前置检查与任务统计口径优化测试用例

## 页面主链路测试

- [ ] 任务页打开后不再展示顶部大标题区域，首屏直接展示任务统计卡片和任务列表。
- [ ] 行业采集任务在“采集结果”列展示“请求 / 成功 / 类目记录”，不展示商品入库口径。
- [ ] 商品采集任务在“采集结果”列展示“原始 / 标准化 / 警告”，兼容旧商品任务。
- [ ] 失败任务展示标准错误类型标签；点击“详情”后展示页面地址、问题类型、建议处理、错误详情。

## API 接口测试

- [ ] `GET /api/jobs` 返回最近 20 条任务，且每条任务包含 `request_count`、`success_count`、`record_count`、`error_type` 字段。
- [ ] 旧数据库没有新增字段时，请求 `/api/jobs` 会自动补齐 `source_jobs` 统计列，不报 SQL 字段缺失。

## 数据库存储与写入测试

- [ ] 表名：`source_jobs`；字段：`request_count`、`success_count`、`record_count`、`error_type`；预期：字段存在，默认值不破坏旧任务读取。
- [ ] 行业采集成功后：`source_jobs.page_type = industry_general`，`request_count = 3`，`success_count` 等于接口成功数，`record_count` 等于行业记录数，`raw_count` 同步记录行业记录数，`normalized_count = 0`。
- [ ] 商品采集成功后：`source_jobs.page_type = hot_products`，`record_count = normalized_count`，`error_type = null`。
- [ ] 采集失败后：`source_jobs.job_status = failed`，`error_type` 写入标准错误类型，`error_message` 保留原始错误摘要。

## 异常与边界测试

- [ ] 未登录或登录过期时，前置检查返回 `ok = false`、`errorType = login_required`。
- [ ] 游客态或权限不足时，前置检查返回 `ok = false`、`errorType = guest_blocked`。
- [ ] Profile 副本不可用或被占用时，前置检查返回 `ok = false`、`errorType = profile_locked`。
- [ ] Chrome 不存在或启动失败时，前置检查返回 `ok = false`、`errorType = browser_blocked`。
- [ ] 页面可打开但未捕获业务接口 Authorization 时，前置检查返回 `ok = false`、`errorType = api_auth_missing`。

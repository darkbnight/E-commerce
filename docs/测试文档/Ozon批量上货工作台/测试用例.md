# Ozon批量上货工作台测试用例

## 1. 测试范围
- 工作台路由 `/ozon-upload`
- 步骤条与操作卡片
- 二级连接配置抽屉
- 本机连接配置保存与刷新自动带出
- 模板装载
- JSON 文件导入入口
- JSON 本地校验
- dry-run 执行
- 导入任务查询
- 类目属性与属性值辅助查询
- 页面关键状态截图

## 2. 执行环境
- 本地 Node.js 22
- 前端构建产物：`frontend/menglar-workbench/dist`
- 后端本地服务：`backend/menglar-workbench-api/server.mjs`
- 浏览器自动化：Playwright Chromium
- Ozon 接口：本地 mock server

## 3. 页面主链路测试
### 用例 3.1 打开页面并填写环境
- 前置条件：前端已构建，后端服务可启动
- 操作步骤：
  1. 打开 `/ozon-upload`
  2. 点击“打开连接配置”
  3. 填入 `Client ID`、`Api Key`、`Base URL`
  4. 点击“保存到本机”
  5. 刷新页面
- 预期结果：
  - 页面正常渲染
  - 顶部显示步骤条
  - 连接状态显示“已就绪”
  - 左侧出现操作卡片与 JSON 编辑区
  - 右侧出现任务查询与类目属性查询模块
  - 刷新后不用重新输入凭证

### 用例 3.2 载入模板并校验
- 前置条件：页面已打开
- 操作步骤：
  1. 点击“载入当前模板”
  2. 点击“本地校验”
- 预期结果：
  - 文本域出现 `items`
  - 结果面板显示“本地校验通过，可以继续 dry-run”

### 用例 3.3 dry-run 模拟执行
- 前置条件：模板已载入且本地校验通过
- 操作步骤：
  1. 点击“仅模拟分片”
- 预期结果：
  - 不请求 Ozon 真实商品接口
  - 页面显示“模拟执行完成，未请求 Ozon”
  - 摘要卡显示条数与分片数

### 用例 3.4 查询导入任务状态
- 前置条件：mock server 提供任务状态接口
- 操作步骤：
  1. 输入 `task_id`
  2. 点击“查询任务”
- 预期结果：
  - 页面出现处理中状态
  - 返回后结果面板显示任务状态

### 用例 3.5 查询类目属性
- 前置条件：mock server 提供类目属性接口
- 操作步骤：
  1. 输入 `category_id`
  2. 点击“查询类目属性”
- 预期结果：
  - 结果面板显示属性返回体

本功能不涉及数据库写入。

## 6. 异常与边界测试
### 用例 6.1 非法 JSON
- 操作步骤：在编辑区输入非法 JSON 后点击“本地校验”
- 预期结果：
  - 结果区提示 JSON 解析错误
  - 不发后端执行请求

### 用例 6.2 缺少凭证执行真实请求
- 操作步骤：不填凭证直接点击“真实执行”
- 预期结果：
  - 按钮不可用或页面展示缺少凭证错误

### 用例 6.3 未确认真实执行
- 操作步骤：不勾选真实执行确认项，点击“真实执行”
- 预期结果：
  - 真实执行按钮不可用

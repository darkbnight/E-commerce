# 萌拉采集任务页交互优化测试用例

## 测试范围

- 页面：`/tasks`
- 功能范围：
  - 任务概览卡片
  - 状态筛选
  - 任务详情展开
  - 成功任务详情查看
  - 失败任务错误详情与复制
- 本轮不修改 API 和数据库结构，仅验证已有任务字段在页面上的回填与展示

## 页面主链路测试

| 用例编号 | 用例名称 | 操作步骤 | 预期结果 | 执行结果 |
|---|---|---|---|---|
| UI-001 | 任务页基础展示 | 打开 `http://127.0.0.1:4186/tasks` | 页面展示采集可用性检查、任务概览卡片、任务列表和筛选按钮 | 已执行，通过 |
| UI-002 | 成功任务显示详情按钮 | 查看成功任务行 | 成功任务操作列展示“详情”按钮 | 已执行，通过 |
| UI-003 | 成功任务详情展开 | 点击成功任务“详情” | 展开任务详情卡片，展示任务类型、任务状态、开始时间、结束时间/耗时、目标网页、采集统计 | 已执行，通过 |
| UI-004 | 失败任务详情保留错误信息 | 点击失败任务“详情” | 展开后仍可看到问题类型、建议处理和错误详情 | 已执行，通过 |
| UI-005 | 失败任务复制入口保留 | 查看失败任务操作列 | 失败任务仍显示“复制”按钮；成功任务不显示“复制”按钮 | 已执行，通过 |
| UI-006 | 详情收起 | 点击已展开任务的“收起” | 详情区收起，列表结构恢复正常 | 已执行，通过 |
| UI-007 | 目标网页链接展示 | 展开任意任务详情 | 目标网页以链接形式展示，可用于后续复盘采集目标 | 已执行，通过 |

## API 接口测试

| 用例编号 | 接口 | 验证点 | 预期结果 | 执行结果 |
|---|---|---|---|---|
| API-001 | `GET /api/jobs` | 页面继续复用原任务接口 | 返回任务数组，包含 `page_url / page_type / job_status / started_at / finished_at` 等详情字段 | 已执行，通过 |

## 数据库存储与写入测试

| 用例编号 | 表名 | 字段级验证 | 预期结果 | 执行结果 |
|---|---|---|---|---|
| DB-001 | `source_jobs` | 抽查任务主键 `id=24`，核验 `page_url / page_type / job_status / started_at / finished_at / raw_count / normalized_count / record_count / error_type / error_message` | 字段存在且页面可据此回填任务详情卡片 | 已执行，通过 |
| DB-002 | `source_jobs` | 抽查失败任务主键 `id=21`，核验 `error_type / error_message / page_url` | 失败任务可展示目标网页和错误详情，不依赖新增字段 | 已执行，通过 |

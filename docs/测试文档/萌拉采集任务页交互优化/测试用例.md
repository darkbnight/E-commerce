# 萌拉采集任务页交互优化测试用例

## 测试范围

- 页面：`/tasks`
- 功能：任务概览、状态筛选、任务表格、错误详情展开、错误复制入口
- 本轮未修改 API、数据库结构和采集逻辑

## 页面主链路测试

| 用例编号 | 用例名称 | 操作步骤 | 预期结果 | 执行结果 |
|---|---|---|---|---|
| UI-001 | 任务页基础展示 | 打开 `http://127.0.0.1:4186/tasks` | 页面展示任务概览卡、任务列表、状态筛选和刷新按钮 | 已执行，通过 |
| UI-002 | 状态标签中文展示 | 查看成功、失败任务行 | 状态显示为“成功”“失败”，不再直接显示英文状态值 | 已执行，通过 |
| UI-003 | 长错误默认收起 | 查看包含长错误堆栈的任务行 | 表格中只显示错误摘要，不撑开整页布局 | 已执行，通过 |
| UI-004 | 错误详情展开 | 点击失败任务的“详情”按钮 | 展开页面地址和完整错误详情，再次点击可收起 | 已执行，通过 |
| UI-005 | 任务状态筛选 | 点击“全部 / 运行中 / 成功 / 失败”筛选按钮 | 表格按任务状态切换展示 | 已执行，通过 |

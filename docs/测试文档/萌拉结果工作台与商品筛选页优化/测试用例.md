# 萌拉结果工作台与商品筛选页优化测试用例

## 页面主链路测试
| 编号 | 用例 | 操作 | 预期 | 执行结果 |
| --- | --- | --- | --- | --- |
| UI-001 | 结果页默认态 | 打开 `/results` | 显示 `结果展示` / `商品筛选` 页签，默认停留在 `结果展示` | 已执行，通过 |
| UI-002 | 默认批次加载 | 打开 `/results` | 自动加载最近一个有商品数据的成功批次，原始结果表格有数据 | 已执行，通过 |
| UI-003 | 当前页加入筛选池 | 在 `结果展示` 点击 `当前页加入筛选池` | 当前页商品进入 `商品筛选` 工作台，接口返回去重统计 | 已执行，通过 |
| UI-004 | 单品加入筛选池 | 在 `结果展示` 表格点击单行 `加入筛选池` | 按钮立即变为 `已在筛选池`，仅该商品进入 `商品筛选` 工作台，不误加入本页其他商品 | 已执行，通过 |
| UI-005 | 商品筛选空态 | 未加入任何商品时切换到 `商品筛选` | 显示空态说明和返回 `结果展示` 入口 | 已执行，通过 |
| UI-006 | 商品筛选全部视图 | 进入 `商品筛选` | 显示 `全部 / 待初筛 / 待测价 / 待找供应链 / 待整理竞品 / 可流转 / 已淘汰` 统计条 | 已执行，通过 |
| UI-007 | 阶段推进主链路 | 在 `商品筛选` 依次点击 `进入测价 -> 测价通过 -> 已找到货源 -> 竞品已整理 -> 进入商品数据整理` | 阶段正确推进，按钮和状态文案同步变化 | 已执行，通过 |
| UI-008 | 刷新后持久化 | 在阶段推进完成后刷新页面 | 工作台状态仍然保留，显示 `已流转商品数据整理` | 已执行，通过 |
| UI-009 | 结果页顶部筛选布局 | 打开 `/results` 默认结果页 | 不展示商品数据统计块，批次和筛选条件位于表格上方，结果表不再被左侧筛选栏压缩 | 已执行，通过 |
| UI-010 | 结果页均价/重量区间筛选 | 在结果页输入均价区间与重量区间 | 表格按 `avg_price_cny` 与 `weight_g` 联合过滤，筛选条件刷新后可回显 | 已补充，待执行 |
| UI-011 | 商品图放大预览 | 点击结果表商品图 | 出现放大预览层，可点击关闭返回表格 | 已执行，通过 |
| UI-012 | 结果表字段排版优化 | 查看结果表首行数据 | 销量/增长合并换行，尺寸/重量换行并位于均价后，潜力指数/销售金额/广告默认不显示小数位 | 已补充，待执行 |
| UI-013 | 原始商品状态展示 | 打开结果页并查看操作列 | 原始商品展示 `待处理 / 已加筛 / 淘汰` 状态，已加筛和淘汰状态有明确按钮反馈 | 已执行，通过 |
| UI-014 | 单品淘汰 | 在结果页点击单行 `淘汰` | 该商品状态变为 `淘汰`，不能再重复加入筛选池 | 已补充，待执行 |
| UI-015 | 当前页全部淘汰 | 在结果页点击 `当前页全部淘汰` | 当前页商品批量写入淘汰状态，接口返回写入/更新统计 | 已补充，待执行 |

## API 接口测试
| 编号 | 接口 | 用例 | 预期 | 执行结果 |
| --- | --- | --- | --- | --- |
| API-001 | `GET /api/result-jobs` | 默认批次列表 | 仅返回成功且存在商品快照的批次 | 已执行，通过 |
| API-002 | `GET /api/products` | 原始商品结果读取 | 返回原始结果页所需字段、筛选项和分页信息 | 已执行，通过 |
| API-003 | `GET /api/product-selection/items` | 工作台列表读取 | 返回工作台条目、来源批次字段和原始商品快照字段 | 已执行，通过 |
| API-004 | `POST /api/product-selection/items` | 单品/批量加入筛选池 | 返回 `insertedCount`、`duplicateCount`、`skippedCount`，且去重生效 | 已执行，通过 |
| API-005 | `PATCH /api/product-selection/items/:id` | 推进阶段与写入测价/供应链/竞品状态 | 返回更新后的工作台条目，非法枚举值返回 400 | 已执行，通过 |
| API-006 | `POST /api/product-selection/items/:id/transfer-to-prep` | 流转到商品数据整理 | 写入流转时间，返回已流转状态 | 已执行，通过 |
| API-007 | `GET /api/products` | 均价区间与重量区间过滤 | 支持 `minAvgPrice/maxAvgPrice/minWeight/maxWeight`，并在 `filters` 中回显 | 已执行，通过 |
| API-008 | `GET /api/products` | 商品状态过滤 | 支持 `productStatus=pending/selected/rejected`，分页总数与状态结果一致 | 已执行，通过 |
| API-009 | `POST /api/product-selection/items` | 淘汰原始商品 | 支持 `selectionStage=screening_rejected`，已存在记录可更新为淘汰并返回 `updatedCount` | 已执行，通过 |

## 数据库存储与写入测试
| 编号 | 表名 | 字段级校验 | 预期 | 执行结果 |
| --- | --- | --- | --- | --- |
| DB-001 | `product_business_snapshots` | `id`、`job_id`、`platform_product_id` | 原始结果页加入筛选池时，来源快照存在且能被正确关联 | 已执行，通过 |
| DB-002 | `product_selection_items` | `source_job_id`、`source_snapshot_id`、`source_platform`、`source_platform_product_id` | 加入筛选池后正确落库，且唯一键防重复写入 | 已执行，通过 |
| DB-003 | `product_selection_items` | `selection_stage`、`pricing_decision`、`initial_cost_price`、`initial_delivery_cost`、`initial_target_price`、`initial_profit_rate` | 测价与利润判断阶段写入正确 | 已执行，通过 |
| DB-004 | `product_selection_items` | `supply_match_status`、`supply_vendor_name`、`supply_reference_url`、`competitor_packet_status` | 找供应链和整理竞品阶段字段写入正确 | 已执行，通过 |
| DB-005 | `product_selection_items` | `transfer_to_prep_at`，并关联 `id` | 流转到商品数据整理时写入非空时间戳，条目仍可回查 | 已执行，通过 |
| DB-006 | `product_selection_items` | 非法 `selection_stage` 请求后查库 | 被拒绝的脏数据不应写入数据库 | 已执行，通过 |
| DB-007 | `product_selection_items` | `source_snapshot_id=2002`、`selection_stage=screening_rejected`、`selection_result=rejected` | 原始商品淘汰写入工作流表，并可被 `product_business_snapshots.id` 回溯 | 已执行，通过 |

## 异常与边界测试
| 编号 | 用例 | 操作 | 预期 | 执行结果 |
| --- | --- | --- | --- | --- |
| ERR-001 | 重复加入筛选池 | 对同一商品重复执行 `POST /api/product-selection/items` | 命中唯一键去重，`duplicateCount` 增加，不产生重复数据 | 已执行，通过 |
| ERR-002 | 非法阶段值拦截 | 调用 `PATCH /api/product-selection/items/:id` 传入非法 `stage` | 接口返回 400，数据库不落脏值 | 已执行，通过 |
| ERR-003 | 商品筛选不混入原始结果 | 未执行加入动作直接切到 `商品筛选` | 显示空态，不自动带入当前批次原始商品 | 已执行，通过 |
| ERR-004 | 单品加入不误伤批量 | 只点击单行 `加入筛选池` 后进入工作台 | 工作台仅出现 1 条数据 | 已执行，通过 |

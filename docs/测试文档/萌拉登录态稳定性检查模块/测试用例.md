## 四、异常与边界测试

### 1. profile 锁文件告警降级
- [x] 已观察到 `Cookies` 文件在复制时出现 `sqlite_backup_failed` / `profile_locked` warning。
- [x] 已确认 `profile.usable=true` 时不再把该 warning 作为阻塞错误。
- [x] 已确认最新 `login-health` 输出中：`profile.errorType=null`、顶层 `ok=true`。

### 2. 剩余风险
- [ ] 紫鸟源 profile 的 `Cookies` 仍可能被系统占用，因此 warning 仍会出现。
- [ ] 当前稳定性依赖 localStorage/sessionStorage 与业务授权请求即可恢复，不依赖 `Cookies` 文件必须成功复制。
- [ ] 若后续萌拉把授权链路改成强依赖 `Cookies`，需要再补针对该文件的专门复制策略。

# Ozon批量上货工具测试用例

## 1. 测试范围
- CLI 模板生成
- 商品数据本地校验
- 批量商品上货分片
- Ozon 导入状态查询
- 价格更新
- 库存更新

## 2. 执行环境
- 本地 Node.js 环境
- 使用 `scripts/test-ozon-batch-tool.mjs` 启动 mock server
- 不连接真实 Ozon 线上环境

## 3. 页面主链路测试
本功能无页面，不涉及页面主链路测试。

## 4. API 接口测试
### 用例 4.1 模板生成
- 前置条件：无
- 操作步骤：执行 `node scripts/ozon-batch-tool.mjs template --output <temp-file>`
- 预期结果：
  - 命令退出码为 0
  - 输出文件存在
  - JSON 顶层包含 `items`

### 用例 4.2 商品数据校验通过
- 前置条件：准备一份合法商品 JSON
- 操作步骤：执行 `node scripts/ozon-batch-tool.mjs validate --input <products.json>`
- 预期结果：
  - 命令退出码为 0
  - 输出 JSON 中 `ok=true`

### 用例 4.3 商品数据校验失败
- 前置条件：准备一份缺少 `images` 或 `attributes` 的商品 JSON
- 操作步骤：执行 `node scripts/ozon-batch-tool.mjs validate --input <invalid-products.json>`
- 预期结果：
  - 命令退出码非 0 或输出 `ok=false`
  - 错误信息能定位缺失字段

### 用例 4.4 商品上货自动分片
- 前置条件：准备 101 条合法商品 JSON，mock server 正常启动
- 操作步骤：执行 `node scripts/ozon-batch-tool.mjs upload --input <products.101.json> --client-id demo --api-key demo --base-url <mock-url>`
- 预期结果：
  - 命令退出码为 0
  - mock server 收到 2 次 `/v2/product/import`
  - 第 1 次 100 条，第 2 次 1 条

### 用例 4.5 导入状态查询
- 前置条件：mock server 正常启动
- 操作步骤：执行 `node scripts/ozon-batch-tool.mjs import-info --task-id 1000 --client-id demo --api-key demo --base-url <mock-url>`
- 预期结果：
  - 命令退出码为 0
  - 返回体包含 `status`

### 用例 4.6 价格更新
- 前置条件：准备合法价格 JSON，mock server 正常启动
- 操作步骤：执行 `node scripts/ozon-batch-tool.mjs prices --input <prices.json> --client-id demo --api-key demo --base-url <mock-url>`
- 预期结果：
  - 命令退出码为 0
  - mock server 收到 `/v1/product/import/prices`

### 用例 4.7 库存更新
- 前置条件：准备合法库存 JSON，mock server 正常启动
- 操作步骤：执行 `node scripts/ozon-batch-tool.mjs stocks --input <stocks.json> --client-id demo --api-key demo --base-url <mock-url>`
- 预期结果：
  - 命令退出码为 0
  - mock server 收到 `/v2/products/stocks`

## 5. 数据库存储与写入测试
本功能不涉及数据库写入。

## 6. 异常与边界测试
### 用例 6.1 输入文件不存在
- 操作步骤：执行 `validate --input missing.json`
- 预期结果：
  - 命令失败
  - 错误信息明确指出文件不存在

### 用例 6.2 缺少 Ozon 凭证
- 操作步骤：执行 `upload --input <products.json>`，不提供 `--client-id` 和 `--api-key`
- 预期结果：
  - 命令失败
  - 错误信息明确指出缺少凭证

### 用例 6.3 dry-run 模式
- 操作步骤：执行 `upload --input <products.json> --dry-run`
- 预期结果：
  - 不请求远程接口
  - 输出分片统计结果

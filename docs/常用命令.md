# 常用命令

启动开发服务  npm run dev

# 基础命令

## 1. 安装依赖

```powershell
npm install
```

## 2. 运行萌拉采集最小链路

```powershell
npm run mvp:probe
```

说明：

- 复用紫鸟运行态会话
- 打开萌拉热销产品页
- 抓取当前页商品数据
- 写入本地 SQLite

## 3. 查看数据库摘要

```powershell
npm run mvp:db:inspect
```

说明：

- 查看最近任务
- 查看原始表和标准化表数量
- 查看样本商品数据

## 4. 启动工作台 API 服务

```powershell
npm run workbench:api
```

说明：

- 默认端口：`4186`
- 服务入口：`backend/menglar-workbench-api/server.mjs`
- 提供：
  - `/api/jobs`
  - `/api/products`
- 构建后也可直接托管工作台静态页面

## 5. 启动工作台前端开发服务

```powershell
npm run workbench:web
```

说明：

- 默认端口：`4178`
- 开发模式，支持前端热更新
- 依赖 `workbench:api` 同时运行

## 6. 构建工作台前端

```powershell
npm run workbench:web:build
```

说明：

- 构建产物输出到：
  - `frontend/menglar-workbench/dist`

## 7. 单服务模式查看工作台

先构建：

```powershell
npm run workbench:web:build
```

再启动：

```powershell
npm run workbench:api
```

访问地址：

- `http://127.0.0.1:4186/tasks`
- `http://127.0.0.1:4186/results`

## 8. 开发模式查看工作台

推荐一键启动：

```powershell
npm run dev
```

说明：

- 同时启动后端 API 和前端 Vite 开发服务
- 后端默认端口：`4186`
- 前端默认端口：`4178`
- 修改 `frontend/menglar-workbench/src/**` 后页面会热更新

如果需要分开启动，也可以使用两个终端。

终端 1：

```powershell
npm run workbench:api
```

终端 2：

```powershell
npm run workbench:web
```

访问地址：

- `http://127.0.0.1:4178/tasks`
- `http://127.0.0.1:4178/results`
- `http://127.0.0.1:4178/shipping-calculator`

## 9. 常用数据文件位置

- SQLite 数据库：
  - `db/menglar-mvp.sqlite`
- 最新运行报告：
  - `data/menglar-mvp/last-run.json`
- 后端服务目录：
  - `backend/menglar-workbench-api/`
- 前端工作台目录：
  - `frontend/menglar-workbench/`

## 10. Ozon 官方物流规则导入

生成本地物流规则：

```powershell
npm run shipping:import
```

校验生成后的规则文件：

```powershell
npm run shipping:validate:generated
```

说明：

- 官方 XLSX 源文件放在 `data/shipping-sources/`
- 生成文件为 `config/shipping/rules.generated.json`
- 确认无误后再人工替换 `config/shipping/rules.json`
- 详细说明见 `data/shipping-sources/README.md`

# 商品筛选测价环节快速定价联动-页面结构与线框方案

## 页面范围

页面仍为 `/results?mode=screening` 的商品筛选工作台，不新增独立页面。

## 商品行结构

```text
商品筛选工作台
┌────────────────────────────────────────────────────────────────────────────┐
│ 商品信息 │ 重量/物流 │ 测价摘要 │ 供应链/竞品 │ 操作                    │
├────────────────────────────────────────────────────────────────────────────┤
│ 图片     │ 重量      │ 预估售价  未测价/¥xx │ 状态     │ 查看竞品详情          │
│ 标题     │ 跨境物流  │ 已记录成本 -/¥xx     │ 供应商   │ 进入测价/重新测价      │
│ 批次     │           │ 利润判断  待确认/通过 │ 竞品     │ 淘汰/流转等            │
└────────────────────────────────────────────────────────────────────────────┘
```

行内测价区只展示摘要，不承担完整填写动作。

## 交互入口

- `进入测价`：打开弹窗，不直接生成或写回测价结论。
- `确认测价通过`：写回测价字段并进入 `source_pending`。
- `确认利润不成立`：写回测价字段并进入 `pricing_rejected`。
- `重新测价`：再次打开弹窗，默认带出上次测价成本和物流成本，可覆盖写回。

## 参数模板

- 下拉选择模板后立即填充模板参数。
- 保存模板时只保存通用参数，不保存当前商品的采购价、重量、尺寸和物流费。
- 模板存储在浏览器本地，服务端和数据库本轮不新增模板表。

## 响应式要求

- 宽屏下弹窗左右布局：左侧参数，右侧结果。
- 窄屏下弹窗改为单列布局，表单字段纵向堆叠。
- 文案和值必须允许换行或压缩，避免遮挡确认按钮。

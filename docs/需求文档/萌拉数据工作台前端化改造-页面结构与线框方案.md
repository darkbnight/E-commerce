# 萌拉数据工作台前端化改造-页面结构与线框方案

## 页面目标
把工作台拆成明确的两个功能页，避免一个页面既看任务又看结果。

## 工作台结构

```text
+----------------------------------------------------------------------------------+
| 侧边导航                                                                         |
| - 采集任务                                                                       |
| - 结果展示                                                                       |
+----------------------+-----------------------------------------------------------+
| 采集任务页           | 结果展示页                                                |
|----------------------|-----------------------------------------------------------|
| 顶部标题             | 顶部标题 + 当前任务卡                                     |
| 任务列表表格         | 统计卡片区                                                |
| 状态 / 时间 / 数量   | 筛选面板                                                  |
| 错误信息             | 商品结果表                                                |
|                      | 分页控件                                                  |
+----------------------+-----------------------------------------------------------+
```

## 采集任务页
- 只关注任务维度
- 包含字段：
  - 任务ID
  - 页面名称
  - 任务状态
  - 开始时间
  - 结束时间
  - 原始数 / 标准化数
  - 警告数
  - 错误信息

## 结果展示页
- 只关注商品结果维度
- 包含区域：
  - 当前任务卡
  - 统计卡片
  - 筛选面板
  - 商品表格

## 交互说明
- 左侧导航切换页面，不混用内容
- 结果页筛选条件改变后，刷新商品接口
- 结果页分页只影响列表，不影响任务页

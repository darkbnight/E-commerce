# 萌拉采集前置检查与任务统计口径优化-页面结构与线框方案

## 页面范围

- 页面：`/tasks`
- 目标：任务页同时能看懂正式采集结果和前置检查失败原因。

## 页面结构

```text
┌──────────────────────────────────────────────────────────────┐
│ 统计卡片区                                                     │
│ [最近任务] [运行中] [成功] [失败]                              │
└──────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────┐
│ 任务列表                                 [全部][运行中][成功][失败][刷新] │
├────────┬────────────┬──────┬────────────┬────────────┬────────────┬──────┤
│ 任务ID │ 任务        │ 状态 │ 时间        │ 采集结果     │ 错误信息     │ 操作 │
├────────┼────────────┼──────┼────────────┼────────────┼────────────┼──────┤
│ #16    │ 萌拉行业数据 │ 成功 │ 开始/结束/耗时 │ 请求3/成功3/子类目30 │ -          │ -    │
│ #15    │ 萌拉行业数据 │ 失败 │ 开始/结束/耗时 │ 请求0/成功0/子类目0  │ 游客态限制   │ 详情 │
└────────┴────────────┴──────┴────────────┴────────────┴────────────┴──────┘
```

## 交互规则

- 点击“详情”：展开当前任务错误详情。
- 再次点击“收起”：关闭详情。
- 点击“复制”：复制完整错误信息。
- 点击状态筛选：只过滤任务列表，不影响统计卡片。

## 不做范围

- 本轮不在页面内提供“重新登录”按钮。
- 本轮不在页面内直接触发采集脚本。
- 本轮不展示完整原始 JSON，只展示任务级摘要。

## 脚本入口说明

页面只负责展示任务状态，不负责解释采集脚本使用方式。固定脚本入口和调用顺序沉淀在：

```text
scripts/menglar-capture/README.md
```

后续人工或 AI 执行采集前，先读取该 README，再按 preflight -> capture 的顺序执行。

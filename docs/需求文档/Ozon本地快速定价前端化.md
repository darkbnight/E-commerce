# Ozon本地快速定价前端化

## 模块一：【需求概述与交互】
- 需求背景：`tools/ozon-pricing-local` 是独立静态工具，入口分散，无法和现有 Menglar Workbench 的 Ozon 上货、物流测算等页面形成统一操作链路。
- 一句话需求描述：把 Ozon 本地快速定价工具迁移到工作台前端页面，保留本地快照数据、自动试算、手动覆盖和 CEL 物流对比能力。
- 解决方案大纲：新增 `/ozon-pricing` 页面；将物流、类目佣金、汇率 JSON 快照复制到前端模块；把原 `app.js` 计算逻辑抽为纯函数；React 页面负责表单状态、结果展示、物流对比弹窗和异常提示。物流费用只作为快速定价估算口径，标准参考以工作台物流计算器为准。
- 用户核心体验链路：打开工作台 -> 点击左侧 `Ozon 快速定价` -> 输入采购成本、重量、类目、物流、毛利和折扣 -> 页面实时刷新售价、利润、计费重量、费用拆分 -> 必要时打开 CEL 物流对比或手动覆盖物流费/佣金率。

## 模块二：【前端与组件设计】
- 新增路由：`/ozon-pricing`。
- 新增页面：`frontend/menglar-workbench/src/pages/OzonPricingPage.jsx`。
- 新增模块目录：`frontend/menglar-workbench/src/modules/ozon-pricing/`。
- 新增纯计算模块：`pricingCalculator.js`，承接计费重、物流费、售价、佣金、利润率、风险提示计算。
- 新增数据快照：`data/logistics.json`、`data/categoryRates.json`、`data/exchangeRates.json`，来源为原本地工具 `tools/ozon-pricing-local/data/`。
- 页面布局：左侧为定价参数表单；右侧为价格结果、核心指标、收入/成本拆分、数据快照信息；弹窗展示 CEL 物流取货点和上门配送费用对比。
- 交互状态：默认态、输入实时计算态、手动覆盖态、物流对比弹窗态、异常参数提示态。

## 模块三：【后端与架构设计】
- API 设计：本轮不新增、不修改 API。
- 数据库设计：本轮不新增表，不写入数据库，不涉及字段迁移。
- 数据来源：沿用原工具静态 JSON 快照，作为前端构建资源导入。
- 埋点设计：本轮不新增埋点。
- 兼容策略：迁移验证完成后删除 `tools/ozon-pricing-local` 原目录；新页面作为工作台内入口，旧目录不再作为维护入口。

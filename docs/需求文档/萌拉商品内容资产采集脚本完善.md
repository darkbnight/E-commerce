# 萌拉商品内容资产采集脚本完善

## 模块一：【需求概述与交互】
### 需求背景
现有 `scripts/menglar-capture` 已经能做登录态检查和热销商品采集，但 `product_content_assets / product_content_skus` 只有表结构，没有稳定的“单商品内容资产采集链路”。结果是数据库虽然能查询内容资产，却没有持续、可复用的上游写入脚本。

### 一句话需求描述
新增一条可直接通过萌拉采集箱商品 ID 获取商品内容资产并入库的稳定脚本链路。

### 解决方案大纲
1. 复用现有登录态检查、浏览器会话和 `source_jobs` 任务记录能力。
2. 进入采集箱页面后复用授权请求头，先调用 `productLibrary/pageQuery` 精确定位商品库记录。
3. 再调用 `improveEditing/{id}` 拉取商品编辑详情，抽取标题、描述、主题标签、主图、SKU 图片与价格等内容资产。
4. 标准化后写入 `product_content_assets / product_content_skus`，按 `content_hash` 去重。

### 用户核心体验链路
1. 用户提供商品 ID。
2. 脚本自动检查登录态并定位采集箱记录。
3. 脚本自动拉取编辑详情并输出结构化内容资产。
4. 数据稳定写入数据库，供后续查询、整理和上货使用。

## 模块二：【前端与组件设计】
本次不新增前端页面，也不修改 React/Vue 组件。

## 模块三：【后端与架构设计】
### 脚本设计
- 新增 `scripts/menglar-capture/content-assets.mjs` 作为单商品内容资产采集入口。
- 新增 `scripts/menglar-capture/lib/content-assets-store.mjs` 负责内容资产标准化、哈希计算与数据库写入。
- 新增 `scripts/test-menglar-content-assets.mjs` 覆盖标准化和入库去重逻辑。

### 数据口径
- 商品身份：`platform + platform_product_id`
- 详情来源：`GET /api/ozon-report-service/v1/improveEditing/{id}`
- 记录定位：`POST /api/ozon-report-service/v1/productLibrary/pageQuery`
- 内容版本去重：`content_hash`

### 数据库存取
- `product_content_assets`：写入标题、描述、标签、主图、产品级图片、内容哈希
- `product_content_skus`：写入 SKU ID、SKU 名称、价格、币种、SKU 图片数组
- `source_jobs`：记录本次单商品内容采集任务状态与结果数量

### 埋点设计
本次不新增埋点。

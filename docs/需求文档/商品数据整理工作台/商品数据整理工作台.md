# 商品数据整理工作台

## 模块一：【需求概述与交互】
### 需求背景
当前工作台已经有“结果展示 / 商品筛选”和“Ozon 上货工具”，但两者之间缺少一个中间整理层。筛选后的商品仍然需要处理标题、描述、类目属性、规格参数、图片、价格库存、物流字段等信息，才能达到可发布上货的状态。如果直接在筛选页或上货页继续堆功能，会让结果查看、人工选品、发布草稿、真实上货的职责混在一起，后续多人开发时容易在同一批前端组件、后端接口和数据库字段上产生冲突。

### 一句话需求描述
新增“商品数据整理”页签和 `/product-data-prep` 子页面，作为筛选商品到 Ozon 上货之间的发布草稿整理工作台。

### 解决方案大纲
1. 侧边栏新增 `商品数据整理` 页签，路由为 `/product-data-prep`。
2. 首版只固定页面入口、职责边界和开发交接说明，不接入真实写入 API。
3. 后续开发时把功能拆成“候选商品导入、字段整理、校验、草稿导出”四个阶段。
4. 数据层不覆盖 `products_normalized` 原始采集/筛选结果，新增发布草稿表承接整理后的字段。
5. 上货工具只消费整理完成的草稿数据，真实发布仍走 Ozon 上货工具的校验与确认流程。

### 用户核心体验链路
1. 用户在 `/results` 的商品筛选页签中筛出候选商品。
2. 用户进入 `/product-data-prep` 查看待整理候选商品。
3. 用户逐个或批量整理商品标题、描述、属性、规格、图片、价格库存和物流字段。
4. 页面提示缺失字段、枚举不合法、图片不合规、尺寸重量异常等问题。
5. 用户将整理完成的商品标记为“发布草稿已就绪”。
6. Ozon 上货工具读取已就绪草稿，继续执行本地校验、dry-run 和真实提交。

### 范围边界
- 本次只新增侧边栏页签、新路由和页面骨架。
- 本次不新增后端 API。
- 本次不新增数据库表。
- 本次不改动 `/results` 商品筛选逻辑。
- 本次不改动 `/ozon-upload` 真实上货执行链路。
- 后续具体开发必须另起功能分支或至少按模块分批提交。

## 模块二：【前端与组件设计】
### 路由设计
- `/product-data-prep`：商品数据整理工作台。

### 页面结构
- 顶部说明区：
  - 页面名：`商品数据整理`
  - 页面职责：承接筛选商品，生成发布草稿。
- 工作流区：
  - `导入筛选商品`
  - `整理发布字段`
  - `发布前校验`
  - `生成上货草稿`
- 安全开发边界区：
  - 标明前后端拆分、接口命名空间、数据库写入边界、mock 联调策略。

### 后续组件拆分建议
- `ProductPrepBatchSelector`
  - 选择候选商品批次或筛选池。
- `ProductPrepTable`
  - 展示待整理商品列表、整理状态、问题数量、草稿状态。
- `ProductPrepEditor`
  - 编辑标题、描述、属性、规格、价格库存、尺寸重量。
- `ProductImageWorkbench`
  - 管理主图、详情图、排序、图片合规状态。
- `ProductPrepValidationPanel`
  - 展示字段缺失、枚举错误、图片问题、物流字段问题。
- `ProductDraftExportPanel`
  - 将已就绪草稿导出或推送给 Ozon 上货工具。

### 前端冲突控制
- 新页面使用独立文件：`frontend/menglar-workbench/src/pages/ProductDataPrepPage.jsx`。
- 首期不要修改 `ResultsPage.jsx` 的大表格逻辑；如需跳转传参，先使用 URL query，不直接抽大组件。
- 上货工具保持只负责执行，不在 `OzonUploadPage.jsx` 内增加数据整理 UI。
- 大组件拆分时按组件文件独立认领，避免多人同时编辑同一文件。

## 模块三：【后端与架构设计】
### API 设计预留
本次不新增 API。后续建议统一使用 `/api/product-data-prep` 命名空间：

- `GET /api/product-data-prep/candidates`
  - 读取筛选后的候选商品。
- `POST /api/product-data-prep/drafts`
  - 从候选商品创建发布草稿。
- `GET /api/product-data-prep/drafts`
  - 查询草稿列表和整理状态。
- `PATCH /api/product-data-prep/drafts/:id`
  - 更新标题、描述、属性、规格、图片、价格库存等草稿字段。
- `POST /api/product-data-prep/drafts/:id/validate`
  - 对单个草稿执行发布前校验。
- `POST /api/product-data-prep/drafts/export`
  - 批量导出给 Ozon 上货工具使用的数据。

### 数据库设计预留
本次不新增数据库。后续建议新增草稿表，不覆盖原始商品表。

建议表：
- `product_publish_drafts`
- `product_publish_draft_images`
- `product_publish_draft_attributes`
- `product_publish_draft_validation_issues`

核心关联：
- `product_publish_drafts.source_job_id -> source_jobs.id`
- `product_publish_drafts.product_normalized_id -> products_normalized.id`
- `product_publish_draft_images.draft_id -> product_publish_drafts.id`
- `product_publish_draft_attributes.draft_id -> product_publish_drafts.id`

字段级验证重点：
- `product_publish_drafts.id`
- `product_publish_drafts.product_normalized_id`
- `product_publish_drafts.draft_status`
- `product_publish_drafts.title`
- `product_publish_drafts.description`
- `product_publish_drafts.category_id`
- `product_publish_drafts.price`
- `product_publish_drafts.stock`
- `product_publish_draft_images.draft_id`
- `product_publish_draft_images.image_url`
- `product_publish_draft_attributes.attribute_id`
- `product_publish_draft_attributes.value`

### 埋点设计预留
- 进入商品数据整理页。
- 从候选商品创建草稿。
- 保存草稿字段。
- 执行草稿校验。
- 草稿校验失败。
- 草稿标记就绪。
- 草稿导出给上货工具。

# 商品筛选竞品详情

## 模块一：【需求概述与交互】
- 需求背景：商品进入筛选工作台后，运营需要回看“结果展示”阶段采集到的竞品经营数据，目前只能在筛选表格中看到少量摘要，判断竞品表现时需要来回切换。
- 一句话需求描述：在商品筛选工作台每个商品行增加“查看竞品详情”，点击后展示该商品在结果展示中采集到的完整经营快照。
- 解决方案大纲：复用 `GET /api/product-selection/items` 已返回的来源商品快照字段，在前端增加详情弹窗，集中展示商品身份、类目、销量、销售额、均价、曝光点击、转化、毛利、广告、尺寸、物流和来源批次。
- 用户的核心体验链路：进入商品筛选 -> 点击行内“查看竞品详情” -> 弹窗查看原始采集数据 -> 关闭后继续当前筛选流程。

## 模块二：【前端与组件设计】
- 修改路由：不新增路由，复用 `/results?mode=screening`。
- 修改组件：`ResultsPage.jsx`
  - `SelectionRow` 操作区增加 `查看竞品详情` 按钮。
  - 新增 `CompetitorDetailDialog` 弹窗组件。
  - 弹窗支持点击遮罩、点击关闭按钮、按 `Escape` 关闭。
- 样式：`styles.css`
  - 新增竞品详情弹窗布局、指标网格和移动端响应式样式。

## 模块三：【后端与架构设计】
- API 设计：不新增、不修改接口。
- 数据来源：继续使用 `GET /api/product-selection/items` 返回的 `item` 对象。
- 数据库 Schema：不新增、不修改数据库字段。
- 埋点设计：本次不新增埋点。

# Ozon批量上货工作台-页面结构与线框方案

## 1. 页面定位
该页面是 Ozon 批量上货的执行页，核心目标不是展示所有 API 能力，而是降低误操作风险，让运营按固定顺序完成批量上货、价格更新或库存更新。

## 2. 页面区域划分
### 顶部 Hero
- 页面标题：`Ozon 批量上货工具`
- 说明：按 `选择动作 -> 导入数据 -> 校验数据 -> 确认执行` 操作
- 连接配置卡片：
  - 状态：已就绪 / 未配置
  - 当前环境：正式 Ozon 或自定义环境
  - 入口：打开连接配置

### 步骤条
- `1 选择动作`
- `2 导入数据`
- `3 校验数据`
- `4 确认执行`

### 主体布局
- 左侧主操作区：承载上货流程。
- 右侧辅助区：承载任务查询、类目属性和属性值查询。

## 3. 左侧结构
### 模块 A：选择操作
- 三张操作卡片：
  - 商品上货
  - 更新价格
  - 更新库存
- 切换卡片时自动载入对应模板，避免模式和 JSON 不一致。

### 模块 B：导入与编辑数据
- `载入当前模板`
- `上传 JSON 文件`
- `清空编辑区`
- `格式化 JSON`
- JSON 编辑区：
  - 当前模式
  - 字符数
  - 条目数
  - 预计分片
  - JSON 是否可解析

### 模块 C：校验与执行
- 连接状态摘要：
  - 是否已保存凭证
  - 当前环境
  - 修改连接配置入口
- `本地校验`
- `仅模拟分片`
- `真实执行`
- 真实执行前必须勾选确认项：
  - 明确当前环境
  - 明确影响条数
- 结果展示：
  - 摘要卡：状态、条数、分片、task_id、警告数、错误数
  - 问题列表：错误与预警
  - 原始 JSON：默认折叠，按需展开

## 4. 右侧结构
### 模块 D：二级连接配置抽屉
- `Client ID`
- `Api Key`
- `Base URL`
- `保存到本机`
- `清除本机配置`
- 说明：配置保存到当前浏览器 `localStorage`，不写入后端数据库。

## 4. 右侧结构
### 模块 E：连接状态
- 当前是否已保存凭证。
- 当前环境地址。
- `打开连接配置`。

### 模块 F：任务查询
- `task_id`
- `查询任务`

### 模块 G：类目属性助手
- `category_id`
- `查询类目属性`
- 若商品 JSON 中存在 `items[0].category_id`，自动带入。

### 模块 H：属性值查询
- `category_id`
- `attribute_id`
- `查询属性值`

### 模块 I：运营提示
- 先查类目属性，补齐必填属性。
- 新商品先 dry-run，再少量真实测试。
- 页面通过不等于 Ozon 审核通过。

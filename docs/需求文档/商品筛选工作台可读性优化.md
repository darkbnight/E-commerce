# 商品筛选工作台可读性优化

## 模块一：【需求概述与交互】

商品筛选工作台原先使用横向表格承载商品、测价、供应链、竞品整理和操作信息。实际浏览时，表格右侧出现大量空白，商品身份和关键决策信息被压缩在左侧，用户需要横向扫很多列才能判断下一步动作。

本次需求是将筛选工作台的商品列表改为“商品决策卡片列表”。每个商品以一行卡片呈现，左侧固定展示商品身份，中间集中展示测价利润、供应链和竞品状态，右侧集中展示操作按钮。

后续体验反馈要求进一步精简顶部区域：去掉入池总数、待测价等统计卡片，将选品工作流过滤区上移到原统计区域，并提升商品筛选工作台内按钮和筛选控件字号。

本轮反馈进一步明确：阶段快捷选择不能取消，需保留按钮式快速切换；卡片内不展示“流转”状态；测价通过时不得覆盖入池时由物流计算器自动测算的运费。

最新体验反馈要求：移除筛选动作成功后的绿色提示条，避免占用列表上方空间；继续放大商品筛选工作台卡片内商品、指标和按钮文字。

核心体验链路：

- 用户进入“商品筛选”。
- 用户直接按卡片读取商品身份、来源批次、测价利润、供应链状态和竞品整理状态。
- 用户优先点击“查看竞品详情”，或根据当前阶段执行“进入测价 / 测价通过 / 已找到货源 / 竞品已整理 / 进入商品数据整理”等操作。

## 模块二：【前端与组件设计】

修改路由：

- 复用现有 `results?mode=screening` 页面，不新增路由。

修改组件：

- `SelectionWorkbenchTable` 保留组件名和父级调用方式，内部从 `<table>` 改为卡片列表。
- `SelectionRow` 从 `<tr>` 改为 `<article className="selection-decision-card">`。
- 商品卡片分为三个主区域：`selection-product-block`、`selection-decision-block`、`selection-action-block`。

样式设计：

- 大屏使用三栏网格：商品身份、决策信息、操作。
- 决策信息内部拆为“测价 / 利润”和“供应链 / 竞品”两个信息组。
- 窄屏自动折叠为单列，避免文字挤压和横向滚动。
- “查看竞品详情”提升为主按钮，其余阶段动作保持次级按钮。
- 移除顶部统计卡片，顶部优先展示筛选条件。
- 筛选模式下主工作区改为单列，不再保留左侧筛选栏。
- 筛选控件和卡片操作按钮字号提升，增强可读性。
- 保留阶段快捷选择按钮条，避免只能通过下拉切换阶段。
- 卡片供应链区域只展示供应链状态和供应商，不展示流转状态。
- 不展示普通成功反馈条，只保留错误反馈。
- 卡片内商品标题、辅助信息、指标标题、指标数值和操作按钮继续放大，优先保证可读性。

## 模块三：【后端与架构设计】

本次不新增后端接口，不修改数据库 Schema，不新增埋点。

数据来源仍复用：

- `GET /api/product-selection/items`
- `product_selection_items`
- `product_business_snapshots`

测价阶段推进规则：

- 入池时 `initialDeliveryCost` 由物流计算器自动测算。
- 点击“测价通过 / 利润不成立”时，前端只补成本、预估售价、利润判断。
- 如果 `initialDeliveryCost` 已存在，阶段推进必须保留原运费，不再用前端简化公式覆盖。
- 只有历史数据缺少 `initialDeliveryCost` 时，才使用前端简化公式作为兜底展示值。

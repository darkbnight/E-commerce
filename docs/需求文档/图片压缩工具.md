# 图片压缩工具

## 模块一：【需求概述与交互】

- 需求背景：商品图片整理时需要把本机商品目录下的图片批量压缩为 JPG，之前只能通过临时命令处理，复用成本高。
- 一句话需求描述：在萌拉数据工作台新增“图片压缩”工具，支持单目录压缩和批量压缩商品子目录，自动把图片压缩为 JPG，并保存到对应目录下的 `压缩图` 子目录。
- 解决方案大纲：封装 Node 脚本复用 `ffmpeg` 执行图片转换；后端提供 `/api/image-compression/compress-jpg`；前端新增 `/image-compression` 页面和左侧入口。
- 用户核心体验链路：打开工作台 -> 点击左侧“图片压缩” -> 输入商品父目录或单个商品目录 -> 选择批量/单目录模式 -> 点击“开始压缩” -> 查看输出目录、压缩数量、体积变化和异常提示。

## 模块二：【前端与组件设计】

- 新增路由：`/image-compression`。
- 新增页面：`ImageCompressionPage`。
- 左侧导航：在 `Ozon 上货工具` 下方新增 `图片压缩` 入口。
- 页面结构：顶部说明区、左侧参数表单、右侧执行结果卡片。
- 表单字段：图片目录、处理模式、输出子目录、JPG 质量、是否覆盖已存在 JPG。
- 辅助说明：页面明示支持 `PNG / JPG / JPEG / WEBP / BMP`，并解释默认质量 `4` 是推荐高质量档。
- 结果展示：目录数量、总图片数、转换数、压缩率、输出体积、各目录输出路径和单图明细。

## 模块三：【后端与架构设计】

- 新增 API：`POST /api/image-compression/compress-jpg`。
- 请求字段：`sourceDir`、`outputDirName`、`quality`、`overwrite`、`mode`。
- 返回字段：`sourceDir`、`mode`、`directoryCount`、`total`、`converted`、`skipped`、`sourceBytes`、`outputBytes`、`compressionRatio`、`supportedExtensions`、`directories`、`items`。
- 脚本复用：`scripts/图片压缩工具/compress-images-to-jpg.mjs` 同时支持 CLI 和 API import。
- 数据库设计：本功能不读写数据库，不新增表和字段。
- 埋点设计：本轮不新增埋点。

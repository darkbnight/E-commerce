# 商品内容资产页面 - 页面结构与线框方案

## 页面目标
这个页面只解决一件事：让用户快速核对某个商品当前有哪些内容版本，以及每个版本下面的 SKU 内容长什么样。

## 页面结构

### 1. 顶部 Hero 区
- 左侧：页面标题“商品内容资产”
- 右侧：3 个摘要卡片
  - 最新版本采集时间
  - 历史版本数量
  - 当前版本 SKU 数量

### 2. 检索区
- 平台选择：当前默认 `ozon`
- 商品 ID 输入框：输入 `platform_product_id`
- 查询按钮
- 重置按钮
- 说明文案：该页查看的是内容资产，不是经营快照

### 3. 示例商品区
- 3 张示例商品卡片
- 点击后直接带入商品 ID 并查询

### 4. 主体双栏区

#### 左栏：历史版本
- 纵向版本卡片列表
- 每张卡片展示：
  - 采集时间
  - 内容哈希缩略
  - SKU 数量
  - 是否为当前版本

#### 右栏上半区：内容详情
- 标题
- 描述
- 标签列表
- 4 个概览指标
- 元信息条
  - 平台
  - 商品 ID
  - 内容哈希
  - 采集时间

#### 右栏中间区：主图 + 图片画廊
- 主图卡片
- 多图画廊卡片

#### 右栏下半区：SKU 资产
- 纵向 SKU 卡片列表
- 每张卡片展示：
  - SKU 名称
  - 平台 SKU ID
  - 价格 + 币种
  - 图片数组

## 关键交互
1. 默认查询后选中最新版本。
2. 点击左侧历史版本卡片，右侧内容详情和 SKU 卡片同时切换。
3. 查询中展示 loading 反馈。
4. 查询无结果时，中栏显示空态，左右栏不显示旧数据。

+---------------------------------------------------------------------------------------------------+

+---------------------------------------------------------------------------------------------------+

## 移动端退化策略
- 三栏改为纵向堆叠：检索区 -> 最新概览 -> 历史版本 -> 内容详情 -> SKU 资产
- 历史版本卡片改为横向信息压缩，避免过长描述

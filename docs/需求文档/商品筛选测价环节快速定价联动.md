# 商品筛选测价环节快速定价联动

## 模块一：【需求概述与交互】

商品筛选工作台的测价动作不能静默替用户下结论。用户希望每个商品保留一个独立测价入口，点开后可以填写采购价、利润目标、费率、物流等参数，也可以套用已保存的参数模板；确认后再生成测价结果并写回商品筛选工作流。

一句话需求：在商品筛选工作台为每个商品提供弹窗式测价入口，复用 Ozon 快速定价计算逻辑，支持参数模板、人工填写、二次确认、重新测价。

解决方案大纲：

- 前端复用 `frontend/menglar-workbench/src/modules/ozon-pricing/pricingCalculator.js` 的 `calculatePricing`。
- 商品行只展示“未测价 / 上次测价摘要”，不再自动直接生成结论。
- 点击“进入测价 / 重新测价”打开测价弹窗。
- 弹窗内支持选择参数模板、填写采购价、毛利目标、折扣、费率、运费、重量尺寸、类目佣金。
- 弹窗右侧实时预览“预估售价、总成本、利润”。
- 用户二次确认“测价通过”或“利润不成立”后，写回现有测价字段并推进阶段。

用户核心体验链路：

1. 用户将商品加入筛选池。
2. 在商品筛选工作台点击某商品的“进入测价”。
3. 弹窗打开，默认带出商品重量、尺寸、跨境物流成本和默认模板。
4. 用户选择模板或手动填写采购价，例如采购价 2 元。
5. 系统复用快速定价逻辑实时预览售价、总成本、利润。
6. 用户确认测价通过或利润不成立。
7. 系统写回测价摘要；后续可点击“重新测价”再次打开弹窗覆盖结果。

## 模块二：【前端与组件设计】

修改路由：无新增路由，仍在 `/results?mode=screening` 内完成。

修改组件：

- `ResultsPage.jsx`
  - 新增测价弹窗状态：当前测价商品、测价表单、用户自定义模板。
  - 新增 `buildPricingFormForEntry(entry, overrides)`，按商品快照和模板生成测价表单。
  - 新增 `computeQuickPricingFromForm(form)`，统一调用快速定价计算模块。
  - 新增 `PricingDialog`，承载模板选择、参数填写、结果预览和二次确认。
  - 行内测价区只展示上次测价摘要；未测价时展示“未测价 / 待确认”。
- `styles.css`
  - 新增测价弹窗、模板栏、参数表单、结果预览样式。

模板口径：

- 内置模板：标准利润 20%、保守测算 25%、投流测试 18%。
- 用户可以将当前弹窗参数保存为本地模板，存储在浏览器 `localStorage`。
- 模板保存的是参数，不保存单个商品的采购价、重量、尺寸和跨境物流费；这些仍从当前商品带出，避免跨商品误用。

## 模块三：【后端与架构设计】

API 设计：

- 不新增 API。
- 继续使用 `PATCH /api/product-selection/items/:id` 写回：
  - `initialCostPrice`
  - `initialDeliveryCost`
  - `initialTargetPrice`
  - `initialProfitRate`
  - `pricingDecision`

数据库设计：

- 不新增表和字段。
- 参数模板暂存浏览器本地，不进入数据库。
- `product_selection_items` 继续保存本次确认后的测价摘要。
- 总成本和利润金额为页面即时展示结果，不单独落库；落库保留成本价、物流成本、预估售价和利润率。

埋点设计：

- 本轮不新增埋点。

## 模块四：【📝 联合自动化验收用例 (TDD核心)】

第一阶段用例（Node 接口与数据库闭环）：

- [ ] 调用 `POST /api/product-selection/items` 将商品加入筛选池，断言返回条目包含 `initialDeliveryCost`。
- [ ] 在弹窗测价确认后调用 `PATCH /api/product-selection/items/:id`，断言返回 `stage=source_pending`、`pricingDecision=continue`、`initialTargetPrice` 为有效数字。
- [ ] 数据库字段级验证：查询表 `product_selection_items`，按 `id` 校验 `selection_stage='source_pending'`、`pricing_decision='continue'`、`initial_delivery_cost > 0`、`initial_target_price > 0`、`source_snapshot_id` 关联来源商品快照。

第二阶段用例（E2E 端到端浏览器闭环）：

- [ ] 在 `/results?mode=result` 将单个商品加入筛选池。
- [ ] 切换到“商品筛选”，断言未测价商品行显示“未测价 / 待确认”。
- [ ] 点击“进入测价”，断言出现测价弹窗。
- [ ] 在弹窗中填写采购价 2 元，断言预览区展示“预估售价、总成本、利润”。
- [ ] 点击“确认测价通过”，断言弹窗关闭，商品阶段进入待找供应链，物流成本不丢失。
- [ ] 点击“重新测价”，断言弹窗可以再次打开并覆盖测价结果。
- [ ] 本轮涉及弹窗和商品行交互变化，需要生成默认态、弹窗测价态、确认结果态截图用于人工复核。

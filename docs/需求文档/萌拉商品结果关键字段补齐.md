# 萌拉商品结果关键字段补齐

## 模块一：【需求概述与交互】

### 需求背景

热销商品采集已能获取清洁抹布类目数据，但商品结果页缺少选品判断需要的关键字段，包括商品图、店铺名称、点击率、广告费、广告占比、平均单价和商品卡创建日期。检查原始响应后确认，萌拉接口已返回大部分字段，主要问题在于标准化入库字段不完整、历史数据未回填、前端未展示。

### 一句话需求描述

补齐热销商品经营快照的关键字段映射、数据库结构、API 返回和商品结果页展示。

### 解决方案大纲

1. 扩展 `product_business_snapshots`
   - 增加商品图、店铺、商品卡创建日期、人民币销售额、平均单价、人民币广告费等字段。
   - 保留已有 `sales_amount`、`ad_cost` 为萌拉返回的卢布口径。

2. 优化热销商品采集映射
   - `skuImg` -> `product_image_url`
   - `shopName` -> `shop_name`
   - `skuName` -> `title`
   - `createDt` -> `product_created_date`
   - `avgPrice` -> `avg_price_rub`
   - `avgPriceRmb` -> `avg_price_cny`
   - `monthGmvRmb` -> `sales_amount_cny`
   - `adsalesRmb` -> `ad_cost_cny`

3. 优化 API 和前端展示
   - `/api/products` 返回新增字段。
   - 关键词支持匹配商品标题和店铺名称。
   - 商品结果表展示图片、标题、店铺、均价、点击率、广告费和商品卡创建日期。

4. 人民币口径原则
   - 优先保存并展示萌拉接口返回的人民币字段。
   - 不在前端用实时汇率重算历史人民币金额，避免历史批次随当前汇率漂移。

### 用户核心体验链路

1. 用户完成热销商品采集。
2. 进入“商品筛选/结果展示”页面。
3. 表格直接看到商品图、标题、店铺、创建日期、均价、点击率和广告费。
4. 用户可以按关键词搜索商品标题、品牌、店铺或类目。

## 模块二：【前端与组件设计】

- 修改 `/results` 页面商品表格。
- 商品信息列展示主图、标题、商品 ID、商品卡创建日期。
- 品牌/店铺列展示品牌、店铺名称、商品类型。
- 均价列展示卢布和人民币。
- 销售金额列展示卢布和人民币。
- 曝光/点击率列展示曝光、点击和点击率。
- 广告列展示广告费、人民币广告费和广告费占比。

- `sales_amount_cny`

- `avg_price_cny`

- 返回新增字段。

- 现有数据库通过迁移脚本从 `products_raw.raw_payload` 回填新增字段。

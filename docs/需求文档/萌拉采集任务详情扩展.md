# 模块一：【需求概述与交互】

## 需求背景

当前数据采集页的任务列表只有失败任务提供“详情”入口。成功任务虽然已经采集完成，但用户无法直接在列表中复盘本次采集的目标网页、任务类型、开始结束时间和统计口径。这会导致后续核对来源、补充字段和追加采集分析时，需要反复查数据库或回到脚本层定位。

## 一句话需求描述

为数据采集页任务列表增加统一的详情展开能力，成功任务和失败任务都可以查看任务详情，其中至少包含目标网页。

## 解决方案大纲

1. 复用现有任务列表数据接口，不新增 API。
2. 将任务列表操作列的“详情”按钮扩展为所有任务可见。
3. 将详情区从“错误详情面板”调整为“任务详情面板”，同时承载成功任务和失败任务信息。
4. 详情区统一展示：
   - 任务类型
   - 任务状态
   - 开始时间
   - 结束时间 / 耗时
   - 目标网页
   - 采集统计
   - 问题类型
   - 建议处理
   - 错误详情（仅异常任务显示）

## 用户核心体验链路

用户进入 `数据采集 -> 任务列表`，无论任务成功还是失败，都可以点击“详情”展开对应任务卡片，直接查看该次采集的目标网页和关键上下文。失败任务依然保留错误复制入口；成功任务则可以用于复盘采集来源和后续字段扩展。

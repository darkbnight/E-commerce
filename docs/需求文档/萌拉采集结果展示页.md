# 萌拉采集结果展示页

## 模块一：【需求概述与交互】
### 需求背景
萌拉采集链路已经可以把热销产品数据写入本地 SQLite，但如果只能靠查库和 JSON 报告看结果，效率太低，也不利于后续接入筛选和工作台能力。

### 一句话需求描述
新增一个本地展示能力，直接读取 `db/menglar-mvp.sqlite` 中最新一次采集结果，展示任务摘要、统计指标和商品列表。

### 解决方案大纲
1. 前端展示页放到 `frontend/menglar-workbench/`。
2. Node 本地服务放到 `backend/menglar-workbench-api/`。
3. 通过只读接口暴露：
   - `GET /api/jobs`
   - `GET /api/products`
4. 前端负责摘要展示、统计卡片、筛选区和商品表格。

### 用户的核心体验链路
1. 用户先运行采集脚本，生成最新 SQLite 数据。
2. 用户启动本地工作台服务。
3. 页面自动读取最新成功任务。
4. 页面展示任务摘要和商品列表。
5. 用户基于关键词、类目、商品类型、销量、销售额做基础筛选。

## 模块二：【前端与组件设计】
### 路由与入口
- 本地访问入口：`http://127.0.0.1:4186/`

### 页面结构
- 顶部摘要区：展示页面标题、任务编号、任务状态、任务时间。
- 统计卡片区：展示商品数、最高销量、最高销售额、平均毛利率。
- 筛选区：关键词、商品类型、一级类目、最低销量、最低销售额、排序方式。
- 结果表格区：按商品维度展示核心指标。

### 关键组件
- `hero`
- `summary-card`
- `filters`
- `table-panel`

## 模块三：【后端与架构设计】
### 服务结构
- 本地 Node HTTP 服务：`backend/menglar-workbench-api/server.mjs`
- 前端工作台：`frontend/menglar-workbench/`
- 数据库：`db/menglar-mvp.sqlite`

### API 设计
- `GET /api/jobs`
  - 返回最近 20 条抓取任务摘要
- `GET /api/products`
  - 默认返回最新成功任务的数据
  - 支持参数：
    - `jobId`
    - `page`
    - `pageSize`
    - `keyword`
    - `productType`
    - `categoryLevel1`
    - `minSales`
    - `minRevenue`
    - `sort`

### 数据库设计
- 本轮不新增表
- 读取现有表：
  - `source_jobs`
  - `products_normalized`

### 埋点设计
- 本轮不新增埋点

## 模块四：【📝 联合自动化验收用例 (TDD核心)】
### 第一阶段用例（Node 接口与数据库闭环）
- [ ] 验证 `GET /api/jobs` 可返回最近任务列表，且 `source_jobs.id`、`job_status`、`raw_count`、`normalized_count` 字段齐全。
- [ ] 验证 `GET /api/products` 默认读取最新成功任务，并查库确认 `products_normalized.job_id` 与返回任务 ID 一致。
- [ ] 验证 `GET /api/products?keyword=Zewa` 可筛出品牌为 `Zewa` 的商品，并查库确认 `products_normalized.brand='Zewa'` 的记录被返回。
- [ ] 验证 `GET /api/products?minSales=20000` 只返回销量大于等于 `20000` 的商品，并查库确认 `products_normalized.sales >= 20000`。

### 第二阶段用例（E2E 端到端浏览器闭环）
- [ ] 启动本地展示页后，浏览器能够正常加载统计卡片和商品表格。
- [ ] 页面输入关键词后，列表只显示匹配数据。
- [ ] 切换排序方式后，结果顺序发生变化且页面无报错。
- [ ] 在无匹配结果时，页面展示空态文案，而不是空白表格。

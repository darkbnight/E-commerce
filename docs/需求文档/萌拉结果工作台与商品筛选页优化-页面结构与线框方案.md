# 萌拉结果工作台与商品筛选页优化-页面结构与线框方案

## 1. 页面目标
本方案用于明确 `/results` 单页面结果工作台的页面结构、主要区域、交互入口和复用边界。

设计原则：
- 去掉结果页顶部纯说明区域，把当前顶部位置改成模式页签。
- 顶部页签只负责切换 `结果展示` 和 `商品筛选`，不承载批次选择。
- 批次选择放到左侧筛选区顶部，作为数据范围筛选的一部分。
- 结果展示和商品筛选在同一个页面内切换，避免跨页面跳转。
- 结果展示是原始数据池，商品筛选是人工筛选池，二者要有明确工作流区分。
- 两个模式复用商品表格基础列，避免维护两套展示逻辑。

## 2. `/results` 单页面结构
### 2.1 页面结构
```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 顶部页签栏                                                                  │
│ [结果展示] [商品筛选]                                                        │
├──────────────────────────────────────────────────────────────────────────────┤
│ 统计条                                                                       │
│ 商品数 10 | 当前命中 10 | 最高销量 47,689 | 最高销售额 ¥10,842,529.28 | 毛利 │
├───────────────┬──────────────────────────────────────────────────────────────┤
│ 筛选条件       │ 商品结果 / 商品筛选池                                        │
│ 数据批次       │ 第 1 / 1 页                                      上一页 下一页 │
│ [#11 商品10]   │ ┌──────────────────────────────────────────────────────────┐ │
│ 关键词         │ │ 结果展示：商品ID 品牌/类型 类目 销量 销售额 转化/毛利... │ │
│ 商品类型       │ │ 商品行                                                   │ │
│ 一级类目       │ │ 商品行                                                   │ │
│ 最低销量       │ │ 结果展示：可将当前页加入筛选池                           │ │
│ 最低销售额     │ │ 商品筛选：只展示筛选池商品，显示候选状态、操作列         │ │
│ 排序方式       │ └──────────────────────────────────────────────────────────┘ │
│ [重置筛选]     │                                                              │
└───────────────┴──────────────────────────────────────────────────────────────┘
```

### 2.2 顶部页签栏
顶部页签栏替代原来的大标题说明区。

内容：
- `结果展示`
- `商品筛选`

职责：
- 只负责切换当前页面模式。
- 不放批次选择。
- 不放长说明文案。
- 不放刷新、导出等二级动作，避免顶部变成杂项工具栏。

展示要求：
- 高度控制在 48px 到 64px。
- 页签状态清晰，当前选中态明显。
- 切换页签时保留当前批次、筛选条件和分页上下文。

### 2.3 左侧筛选区
左侧筛选区是当前页面的主要控制区。

字段顺序：
1. 数据批次
2. 关键词
3. 商品类型
4. 一级类目
5. 最低销量
6. 最低销售额
7. 排序方式
8. 当前结果数量
9. 重置筛选

数据批次放在筛选区顶部，因为它决定后续所有统计和商品表格的数据范围。

## 3. 批次选择器
### 3.1 触发位置
批次选择器从顶部工具栏移出，放到左侧筛选区的 `数据批次` 字段中。

默认展示：
```text
数据批次
┌─────────────────────┐
│ #11 · 商品 10        │
│ 热销商品 · 04-18     │
└─────────────────────┘
```

点击后弹出批次列表或下拉面板。

### 3.2 默认列表规则
- 只展示 `job_status='success'`。
- 只展示 `product_count > 0`。
- 不展示失败、运行中、空商品数量批次。

### 3.3 批次列表结构
```text
┌─────────────────────────────┐
│ 选择数据批次                 │
│ [ ] 显示空批次和失败批次      │
├─────────────────────────────┤
│ #11                          │
│ 商品 10 · 热销商品 · 04-18   │
├─────────────────────────────┤
│ #8                           │
│ 商品 25 · 热销商品 · 04-17   │
└─────────────────────────────┘
```

批次列表只展示必要信息：
- 批次编号
- 商品数量
- 任务类型
- 结束时间

不展示长 URL、错误栈、完整任务描述。

### 3.4 显示不可用批次
当用户开启“显示空批次和失败批次”：
- 空商品批次显示为弱化状态。
- 失败批次显示失败状态。
- 运行中批次显示运行中状态。
- 默认不建议选择失败批次；若允许选择，右侧内容区展示失败或空态，不展示伪商品数据。

### 3.5 空商品批次状态
当用户显式选择 `product_count=0` 的批次时：
```text
┌──────────────────────────────────────────────┐
│ 当前批次没有商品明细数据                       │
│ #16 是成功任务，但没有关联到 products_normalized 商品记录。 │
│ [切换到最近有商品数据的批次 #11] [重新选择批次] │
└──────────────────────────────────────────────┘
```

空态要求：
- 明确说明“任务存在但无商品数据”。
- 不把该状态表达成系统错误。
- 提供回到最近有效批次的入口。

## 4. 结果展示模式
### 4.1 页面结构
```text
顶部页签：结果展示选中
统计条：商品数 / 当前命中 / 最高销量 / 最高销售额 / 平均毛利率
左侧：批次选择 + 筛选条件
右侧：原始采集商品结果表格 + 加入筛选池动作
```

### 4.2 商品表格
结果展示模式中，表格用于浏览原始采集结果，不承担人工候选判断。

主操作：
- `将当前页加入筛选池`
- 第一阶段按当前页 `items` 加入筛选池，避免一次性加入过大结果集。
- 重复加入同一商品时不重复计数。

列：
- 商品 ID
- 品牌 / 类型
- 类目
- 销量
- 销量增长
- 潜力指数
- 销售额
- 曝光 / 点击
- 转化 / 毛利
- 物流 / 时效
- 尺寸 / 重量

优化点：
- 表头固定。
- 表格内部滚动。
- 行高压缩。
- 数字列右对齐。
- 毛利率负数标红，正数使用强调色。
- 商品 ID 保持等宽字体。

## 5. 商品筛选模式
### 5.1 页面结构
```text
顶部页签：商品筛选选中
筛选状态条：筛选池 10 | 待判断 8 | 已加入候选 2 | 已排除 0
左侧：批次选择 + 筛选条件
右侧：筛选池商品表格
```

### 5.2 筛选池规则
- 商品筛选模式不默认展示当前批次全部商品。
- 商品筛选模式只展示用户从结果展示模式加入筛选池的商品。
- 筛选池为空时展示引导：`请先在结果展示中筛选商品，并点击“将当前页加入筛选池”。`
- 筛选池第一阶段只保存在前端内存中，刷新页面会丢失。

### 5.3 与结果展示模式的复用关系
复用：
- 批次选择器
- 商品统计条
- 筛选条件面板
- 商品表格基础列
- 分页逻辑
- API 查询参数组装逻辑

商品筛选模式新增：
- 候选状态。
- 行操作按钮。
- 候选统计。
- 批量操作预留入口。

### 5.4 第一阶段临时筛选状态
第一阶段不落库，页面内维护临时状态。

状态：
- `pending`：待判断。
- `candidate`：已加入候选。
- `rejected`：已排除。

限制：
- 刷新页面后临时状态丢失。
- 最终候选结果不能作为长期工作流依据。
- 如果后续要接上架流程，必须新增候选商品表。

### 5.5 后续落库版本预留
当用户确认筛选流程有效后，商品筛选模式升级为正式候选池：
- `加入候选` 写入 `product_candidates`。
- `排除` 写入 `product_candidates`。
- 支持备注、导出、后续上架分析。

## 6. 组件层级
```text
ResultsPage
├── ResultModeTabs
├── ProductMetricsBar
├── ProductWorkbenchLayout
│   ├── ProductFilterPanel
│   │   └── ResultBatchSelector
│   └── ProductResultTable mode="readonly | screening"
│       └── ProductScreeningActions
```

说明：
- 不新增 `ProductScreeningPage`。
- `ResultsPage` 根据 `mode` 切换表格模式和筛选状态条。
- 批次选择器始终由 `ProductFilterPanel` 承载。

## 7. 数据流
### 7.1 页面初始化
```text
进入 /results
  -> 读取 URL mode，默认 mode=result
  -> GET /api/result-jobs
  -> 默认选择最新可用批次
  -> GET /api/products?jobId=当前批次
  -> 展示统计、筛选条件、商品表格
  -> 用户点击“将当前页加入筛选池”
```

### 7.2 模式切换
```text
点击顶部页签
  -> 更新 URL mode
  -> 保留当前 jobId 和筛选条件
  -> 不重新选择批次
  -> 商品筛选模式只展示筛选池商品
```

### 7.3 批次切换
```text
在左侧筛选区点击数据批次
  -> 选择新批次
  -> 重置页码到第 1 页
  -> GET /api/products?jobId=新批次
  -> 当前 mode 不变
```

## 8. 响应式要求
桌面端：
- 左侧筛选栏固定宽度。
- 右侧表格占主要宽度。
- 表格横向可滚动。

窄屏：
- 顶部页签保持在页面顶部。
- 筛选栏折叠到表格上方。
- 表格保持横向滚动，不强行压缩所有列。

## 9. 验收关注点
- 删除顶部说明后，首屏应直接看到页签、批次筛选和商品表格。
- 顶部区域只展示 `结果展示` / `商品筛选` 页签。
- 批次选择器位于左侧筛选区顶部。
- 默认批次不能再被空商品批次抢占。
- 批次选择器默认列表不能混入失败批次和空商品批次。
- 结果展示和商品筛选复用同一套商品展示组件。
- 商品筛选状态应清晰，不与结果展示只读状态混淆。

# 萌拉结果工作台与商品筛选页优化-页面结构与线框方案

## 1. 页面目标
本方案用于明确 `/results` 和 `/product-screening` 的页面结构、主要区域、交互入口和复用边界。

设计原则：
- 去掉结果页顶部纯说明区域，把首屏空间交给批次选择、筛选和表格。
- 批次选择是结果浏览的一级能力，不能隐藏在任务页。
- 结果页只负责看批次数据，商品筛选页负责人工选品动作。
- 两个页面尽量复用商品筛选条件和商品表格，避免维护两套展示逻辑。

## 2. `/results` 结果展示页
### 2.1 页面结构
```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 顶部工具栏                                                                  │
│ [结果展示] [批次 #11 · 商品 10] [切换批次] [刷新] [进入商品筛选]             │
├──────────────────────────────────────────────────────────────────────────────┤
│ 统计条                                                                       │
│ 商品数 10 | 当前命中 10 | 最高销量 47,689 | 最高销售额 ¥10,842,529.28 | 毛利 │
├───────────────┬──────────────────────────────────────────────────────────────┤
│ 筛选条件       │ 商品结果                                                     │
│ 关键词         │ 第 1 / 1 页                                      上一页 下一页 │
│ 商品类型       │ ┌──────────────────────────────────────────────────────────┐ │
│ 一级类目       │ │ 表头固定：商品ID 品牌/类型 类目 销量 销售额 转化/毛利... │ │
│ 最低销量       │ │ 商品行                                                   │ │
│ 最低销售额     │ │ 商品行                                                   │ │
│ 排序方式       │ │ 商品行                                                   │ │
│ [重置筛选]     │ └──────────────────────────────────────────────────────────┘ │
└───────────────┴──────────────────────────────────────────────────────────────┘
```

### 2.2 顶部工具栏
顶部工具栏替代原来的大标题说明区。

内容：
- 页面短标题：`结果展示`
- 当前批次摘要：`#11 · 热销商品 · 商品 10`
- `切换批次` 按钮
- `刷新` 按钮
- `进入商品筛选` 按钮

展示要求：
- 高度控制在 56px 到 72px。
- 不展示长说明文案。
- 当前批次摘要必须一眼能看出批次编号和商品数量。

### 2.3 批次选择面板
触发方式：
- 点击顶部工具栏的 `切换批次`。

默认列表规则：
- 只展示 `job_status='success'`。
- 只展示 `product_count > 0`。
- 不展示失败、运行中、空商品数量批次。

面板结构：
```text
┌─────────────────────────────┐
│ 选择结果批次                 │
│ [ ] 显示空批次和失败批次      │
├─────────────────────────────┤
│ #11    商品 10               │
│ 热销商品 · 04-18 00:17       │
├─────────────────────────────┤
│ #8     商品 25               │
│ 热销商品 · 04-17 21:02       │
└─────────────────────────────┘
```

当用户开启“显示空批次和失败批次”：
- 空商品批次显示为弱化状态。
- 失败批次显示失败状态，不建议默认可选；若可选，进入后展示错误或空态，不展示商品表伪数据。

### 2.4 空商品批次状态
当用户显式选择 `product_count=0` 的批次时：
```text
┌──────────────────────────────────────────────┐
│ 当前批次没有商品明细数据                       │
│ #16 是成功任务，但没有关联到 products_normalized 商品记录。 │
│ [切换到最近有商品数据的批次 #11] [重新选择批次] │
└──────────────────────────────────────────────┘
```

空态要求：
- 明确说明“任务存在但无商品数据”。
- 不把该状态表达成系统错误。
- 提供回到最近有效批次的入口。

### 2.5 筛选条件区
保留在左侧，密度提高。

字段：
- 关键词：商品 ID / 品牌 / 类目
- 商品类型
- 一级类目
- 最低销量
- 最低销售额
- 排序方式
- 当前结果数量
- 重置筛选

交互：
- 修改任意条件后回到第 1 页。
- 重置筛选只重置筛选条件，不切换当前批次。

### 2.6 商品表格区
表格应成为页面主区域。

优化点：
- 表头固定。
- 表格内部滚动。
- 行高压缩。
- 数字列右对齐。
- 毛利率负数标红，正数使用强调色。
- 商品 ID 保持等宽字体。
- 操作按钮不出现在结果页表格中，避免结果核验页变成工作流页。

## 3. `/product-screening` 商品筛选页
### 3.1 页面结构
```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 顶部工具栏                                                                  │
│ [商品筛选] [批次 #11 · 商品 10] [切换批次] [返回结果页] [导出候选(后续)]     │
├──────────────────────────────────────────────────────────────────────────────┤
│ 筛选状态条                                                                  │
│ 全部 10 | 未处理 8 | 已加入候选 2 | 已排除 0                                 │
├───────────────┬──────────────────────────────────────────────────────────────┤
│ 筛选条件       │ 商品筛选表格                                                 │
│ 同结果页       │ [候选状态筛选：全部 / 未处理 / 已加入候选 / 已排除]            │
│               │ ┌──────────────────────────────────────────────────────────┐ │
│               │ │ 选择 商品ID 品牌 类目 销量 销售额 毛利 状态 操作          │ │
│               │ │ □ 商品行                               [加入候选] [排除] │ │
│               │ │ □ 商品行                               [加入候选] [排除] │ │
│               │ └──────────────────────────────────────────────────────────┘ │
└───────────────┴──────────────────────────────────────────────────────────────┘
```

### 3.2 与结果页的复用关系
复用：
- 批次选择器
- 商品统计条
- 筛选条件面板
- 商品表格基础列
- 分页逻辑
- API 查询参数组装逻辑

商品筛选页新增：
- 行选择框。
- 候选状态。
- 行操作按钮。
- 候选统计。
- 批量操作预留入口。

### 3.3 第一阶段临时筛选状态
第一阶段不落库，页面内维护临时状态。

状态：
- `pending`：未处理。
- `candidate`：已加入候选。
- `rejected`：已排除。

限制：
- 刷新页面后临时状态丢失。
- 最终候选结果不能作为长期工作流依据。
- 如果后续要接上架流程，必须新增候选商品表。

### 3.4 后续落库版本预留
当用户确认筛选流程有效后，商品筛选页升级为正式候选池：
- `加入候选` 写入 `product_candidates`。
- `排除` 写入 `product_candidates`。
- 支持备注、导出、后续上架分析。

```text
ResultsPage
├── ResultToolbar
│   └── ResultBatchSelector

ProductScreeningPage

    └── ProductScreeningActions
```

## 5. 数据流
### 5.1 结果页
```text
进入 /results
  -> GET /api/result-jobs
  -> 默认选择最新可用批次
  -> GET /api/products?jobId=当前批次
  -> 展示统计、筛选条件、商品表格
```

### 5.2 商品筛选页
```text
从 /results 点击进入商品筛选
  -> 继承 URL 中 jobId 和筛选条件
  -> GET /api/products?jobId=当前批次&筛选条件
  -> 展示同一批商品
  -> 用户执行加入候选/排除
  -> 第一阶段只更新前端临时状态
```

## 6. 响应式要求
桌面端：
- 左侧筛选栏固定宽度。
- 右侧表格占主要宽度。
- 表格横向可滚动。

窄屏：
- 顶部工具栏换行。
- 筛选栏折叠到表格上方。
- 表格保持横向滚动，不强行压缩所有列。

## 7. 验收关注点
- 删除顶部说明后，首屏应直接看到批次、筛选和商品表格。
- 默认批次不能再被空商品批次抢占。
- 批次选择器默认列表不能混入失败批次和空商品批次。
- 结果页和商品筛选页复用同一套商品展示组件。
- 商品筛选页的人工筛选状态应清晰，不与结果页只读状态混淆。

# 萌拉结果工作台与商品筛选页优化-页面结构与线框方案

## 1. 页面目标
本方案用于明确 `/results` 中 `结果展示` 和 `商品筛选` 两个阶段的页面结构、区域职责和流转关系。

本轮的核心不是做两个样式页签，而是把两个不同业务阶段拆清楚：

- `结果展示`：看原始批次数据，并把目标商品送入筛选池
- `商品筛选`：对已入池商品做选品决策并流转下游

## 2. 总体结构
### 2.1 页面入口结构
```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 顶部阶段页签                                                                │
│ [结果展示] [商品筛选]                                                        │
├──────────────────────────────────────────────────────────────────────────────┤
│ 对应阶段工作区                                                               │
│ 结果展示：批次原始数据浏览区                                                 │
│ 商品筛选：入池商品选品决策区                                                 │
└──────────────────────────────────────────────────────────────────────────────┘
```

### 2.2 顶部页签设计原则
- 页签不是简单视图切换，而是业务阶段切换。
- 切换到 `商品筛选` 后，页面结构、筛选条件和主操作都应发生变化。
- 顶部统计里的“入池总数”对应“全部”视图，不是单独阶段。

## 3. 结果展示
### 3.1 页面定位
`结果展示` 只负责原始数据呈现，不负责后续选品决策。

页面回答的问题是：
`这个批次里有哪些商品值得送去后续筛选？`

### 3.2 结构草图
```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 顶部页签：[结果展示] [商品筛选]                                              │
├──────────────────────────────────────────────────────────────────────────────┤
│ 统计条：商品数 | 当前命中 | 最高销量 | 最高销售额 | 平均毛利                 │
├───────────────┬──────────────────────────────────────────────────────────────┤
│ 左侧控制区     │ 右侧原始结果区                                                │
│ 数据批次       │ 标题：商品结果                                                │
│ 关键词         │ 副标题：第 N / N 页                                           │
│ 商品类型       │ [当前页加入筛选池] [上一页] [下一页]                          │
│ 一级类目       │ ┌──────────────────────────────────────────────────────────┐ │
│ 最低销量       │ │ 原始结果表格                                              │ │
│ 最低销售额     │ │ 每行都有 [加入筛选池]                                    │ │
│ 排序方式       │ │ 商品信息 | 品牌店铺 | 类目 | 销量 | 销售额 | 毛利...     │ │
│ [重置筛选]     │ └──────────────────────────────────────────────────────────┘ │
└───────────────┴──────────────────────────────────────────────────────────────┘
```

### 3.3 结果展示的加入入口
必须同时存在两种入口：

- `当前页加入筛选池`
  - 适合批量推进当前页全部命中商品
- `单品加入筛选池`
  - 适合只挑部分商品进入后续流程

关键要求：
- 单品加入时不能误加入当前页其他商品
- 已经在筛选池中的商品要去重提示

### 3.4 左侧控制区
字段顺序：
1. 数据批次
2. 关键词
3. 商品类型
4. 一级类目
5. 最低销量
6. 最低销售额
7. 排序方式
8. 当前结果数
9. 重置筛选

说明：
- `数据批次` 是这个阶段的一级控制项
- 所有筛选条件都服务于“缩小原始结果范围”
- 不在这里放供应链、测价、利润判断之类的字段

## 4. 商品筛选
### 4.1 页面定位
`商品筛选` 是选品决策工作台，不是临时候选池列表。

页面回答的问题是：
`这个商品值不值得做，如果值得做，现在推进到哪一步了？`

### 4.2 业务阶段
商品进入 `商品筛选` 后，处理链路固定为：

1. 初步筛选
2. 测价并判断是否值得做
3. 寻找供应链条
4. 整理竞品数据
5. 交给商品数据整理

说明：
- “测价”和“利润判断”视为一个过程
- 只有利润空间成立后，商品才继续进入找供应链和整理竞品阶段

### 4.3 结构草图
```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 顶部页签：[结果展示] [商品筛选]                                              │
├──────────────────────────────────────────────────────────────────────────────┤
│ 状态统计条：全部 | 待初筛 | 待测价 | 待找供应链 | 待整理竞品 | 可流转        │
├───────────────┬──────────────────────────────────────────────────────────────┤
│ 左侧过滤区     │ 右侧选品决策区                                                │
│ 阶段状态       │ 标题：商品筛选工作台                                          │
│ 是否已测价     │ 副标题：当前展示已入池商品                                    │
│ 是否利润成立   │ [批量标记] [进入商品数据整理]                                 │
│ 是否已找货源   │ ┌──────────────────────────────────────────────────────────┐ │
│ 是否已整理竞品 │ │ 商品行：基础信息 + 来源 + 测价利润 + 供应链 + 阶段 + 操作 │ │
│ 类目/品牌      │ │ 商品行                                                    │ │
│ 来源批次       │ │ 商品行                                                    │ │
│ [重置过滤]     │ └──────────────────────────────────────────────────────────┘ │
└───────────────┴──────────────────────────────────────────────────────────────┘
```

### 4.4 “全部”视图
顶部的“入池总数”本质上就是“全部”视图，承担两个作用：

- 查看所有进入筛选池商品的当前状态分布
- 作为总览入口，避免用户只能按单一阶段逐个查看

因此这里建议直接产品化为：
- `全部`
- `待初筛`
- `待测价`
- `待找供应链`
- `待整理竞品`
- `可流转`

### 4.5 左侧过滤区
这里不再复用 `结果展示` 的原始筛选条件。

建议字段：
1. 阶段状态
2. 是否已完成测价
3. 是否利润成立
4. 是否已找供应链
5. 是否已整理竞品数据
6. 类目 / 品牌
7. 来源批次
8. 重置过滤

说明：
- `来源批次` 退化为辅助过滤项
- 左侧控制项服务于工作推进，不服务于原始结果缩小范围

### 4.6 右侧选品决策区
单行商品至少要能看到：
- 商品基础信息
- 来源批次信息
- 当前处理阶段
- 测价结果
- 利润判断结果
- 供应链匹配情况
- 竞品数据整理状态
- 流转下游动作

建议操作：
- 标记淘汰
- 记录测价结果
- 标记利润成立 / 不成立
- 补充供应链
- 标记竞品数据已整理
- 流转商品数据整理

## 5. 两个阶段的主视角差异
### 5.1 对照表
| 维度 | 结果展示 | 商品筛选 |
| --- | --- | --- |
| 主视角 | 批次 | 商品 |
| 目标 | 看原始结果 | 做选品决策 |
| 核心动作 | 当前页加入 / 单品加入筛选池 | 测价判断、找供应链、整理竞品、流转下游 |
| 左侧控制 | 原始数据筛选 | 工作流过滤 |
| 主要字段 | 经营数据 | 阶段状态、测价、利润、供应链、竞品整理 |
| 批次角色 | 一级控制项 | 回溯信息 / 辅助过滤 |

## 6. 状态流线框
### 6.1 单商品阶段流转
```text
结果展示
  -> 当前页加入筛选池 / 单品加入筛选池
  -> 商品筛选 / 待初筛
  -> 商品筛选 / 待测价与利润判断
  -> 商品筛选 / 待找供应链
  -> 商品筛选 / 待整理竞品数据
  -> 商品筛选 / 可流转商品数据整理
  -> 商品数据整理
```

### 6.2 淘汰支路
```text
商品筛选 / 待初筛
  -> 初筛淘汰
  -> 停止流转

商品筛选 / 待测价与利润判断
  -> 利润不成立
  -> 停止流转
```

## 7. 交互细节
### 7.1 从结果展示进入商品筛选
- 用户在 `结果展示` 中可按页加入，也可单品加入
- 若筛选池为空，`商品筛选` 展示引导空态
- 若筛选池非空，默认展示“全部”或“待初筛”视图

### 7.2 商品筛选内的操作反馈
- 阶段切换后顶部统计条同步刷新
- 已完成测价并利润成立的商品，在列表中有明确视觉标识
- 只有满足要求的商品才出现“进入商品数据整理”主操作

### 7.3 空态设计
`商品筛选` 的空态至少区分：
- 尚未加入任何商品
- 已有商品，但当前过滤条件下为空
- 当前阶段无待处理商品

## 8. 验收关注点
- `结果展示` 必须同时支持当前页加入和单品加入
- `商品筛选` 必须体现“工作推进”，不能只是再看一次原始表
- “全部”视图要明确承担总览作用
- “测价”和“利润判断”必须合并表达
- 只有利润成立的商品才能进入找供应链和整理竞品阶段
- 批次在 `商品筛选` 中只能作为来源信息和辅助过滤，不再作为主结构

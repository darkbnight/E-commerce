# 萌拉采集结果展示页-页面结构与线框方案

## 页面目标
让用户不查库也能直接看到最新一次萌拉采集结果，并对结果做基础筛选。

## 页面结构

```text
+-----------------------------------------------------------------------------------+
| 页面标题 / 最新任务信息 / 任务状态                                                 |
+-----------------------------------------------------------------------------------+
| 统计卡片1 | 统计卡片2 | 统计卡片3 | 统计卡片4                                     |
+-----------------------------------------------------------------------------------+
| 左侧筛选栏                              | 右侧商品结果表                           |
|-----------------------------------------|------------------------------------------|
| 关键词                                  | 表头：商品ID / 品牌 / 类目 / 销量 ...    |
| 商品类型                                | 行数据：最新任务的商品指标               |
| 一级类目                                |                                          |
| 最低销量                                | 分页控件                                 |
| 最低销售额                              |                                          |
| 排序方式                                |                                          |
| 应用筛选 / 重置                         |                                          |
+-----------------------------------------------------------------------------------+
```

## 关键区域说明

### 1. 顶部英雄区
- 左侧：标题、副标题
- 右侧：最新任务号、任务状态、入库数量

### 2. 统计卡片区
- 商品总数
- 最高销量
- 最高销售额
- 平均毛利率

### 3. 左侧筛选栏
- 关键词
- 商品类型
- 一级类目
- 最低销量
- 最低销售额
- 排序方式

### 4. 商品结果表
- 展示当前筛选条件下的数据
- 每行至少包含：
  - 商品ID
  - 品牌 / 商品类型
  - 类目
  - 销量
  - 销量增长
  - 潜力指数
  - 销售额
  - 曝光 / 点击
  - 转化 / 毛利
  - 物流 / 时效
  - 尺寸 / 重量

## 交互说明
- 页面默认自动读取最新成功任务
- 用户点击“应用筛选”后刷新表格
- 用户点击“重置”后恢复默认筛选
- 用户切换分页后继续沿用当前筛选条件

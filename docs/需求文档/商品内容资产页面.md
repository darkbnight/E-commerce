# 商品内容资产页面

## 模块一：【需求概述与交互】
当前页面原本以单商品查询和单商品详情展示为主，筛选、版本列表、内容字段、图片、SKU 和经营快照混在同一屏，导致信息重复且不适合作为“已采集内容资产”的浏览入口。

一句话需求：将商品内容资产页面优化为“内容资产浏览器”，默认按商品展示资产列表，点击某一行后在右侧详情抽屉查看该商品的内容、图片、SKU、版本和经营快照。

解决方案大纲：
- 主页面展示筛选区、汇总指标和商品资产表格。
- 表格每行代表一个平台商品，展示主图、标题、平台商品 ID、状态、图片数、SKU 数、标签数、版本数和最新采集时间。
- 商品详情从主页面移入右侧抽屉，抽屉内按 `概览 / 内容 / 图片 / SKU / 版本 / 经营` Tab 分组。
- 后端 `GET /api/product-content` 支持不传 `productId` 时返回列表模式：每个商品最新一条内容版本。

用户核心体验链路：
1. 用户进入“商品内容资产浏览器”。
2. 通过平台、商品 ID/标题关键词、内容状态、历史版本筛选商品。
3. 在表格中快速判断哪些商品缺图、缺 SKU、标签不足或待确认。
4. 点击某行打开右侧详情抽屉。
5. 在抽屉中切换内容、图片、SKU、版本、经营快照完成核查。

  - 新增浏览器顶部、筛选栏、指标条、固定表格、缩略图、状态标签和详情抽屉样式。

页面状态：
- 默认态：展示列表、筛选区、指标条。
- 加载态：列表接口读取中。
- 结果态：表格展示内容资产。
- 详情态：右侧抽屉打开。
- 空筛选态：表格显示“没有符合筛选条件的内容资产”。

## 模块三：【后端与架构设计】
修改 API：
- `GET /api/product-content`
  - 传 `productId`：保持原能力，读取指定商品最新内容版本或历史版本。
  - 不传 `productId`：新增列表模式，返回当前平台下每个商品最新一条内容资产。

列表模式返回字段：
- `query.platform`
- `items[]`
- `total`
- `items[].sku_count`
- `items[].version_count`

数据库读取：
- 主表：`product_content_assets`
- 子表：`product_content_skus`
- 列表模式通过 `NOT EXISTS` 选出同一 `platform + platform_product_id` 下 `captured_at` 与 `id` 最新的版本。
- `sku_count` 使用 `COUNT(DISTINCT product_content_skus.id)`。
- `version_count` 使用同商品历史版本计数子查询。

本轮不新增数据库表，不新增埋点。

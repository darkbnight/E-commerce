# 商品内容资产页面

## 模块一：【需求概述与交互】
### 需求背景
当前后端已经具备商品内容资产与 SKU 资产的最小读取能力，但工作台里还没有一个可视化页面去查看采集后的标题、描述、标签、图片和 SKU 内容效果。现阶段如果要核对内容资产，只能查库或直接请求接口，验证成本高，也不利于后续确认采集质量和内容版本变化。

### 一句话需求描述
新增一个“商品内容资产”页面，支持按平台商品 ID 查看某个商品的最新内容资产、历史版本、对应 SKU 内容效果，以及最近一次经营快照。

### 解决方案大纲
1. 在工作台新增 `/product-content` 页面和导航入口。
2. 页面接入现有内容资产读取接口，不新增新的后端接口口径。
3. 页面分为检索区、示例商品区、版本列表区、内容概览区、经营快照区、SKU 资产区，先满足查看与核对，不做编辑。
4. 默认展示最新版本；切换历史版本时，联动展示该版本对应的 SKU 列表和图片。
5. 提供可直接点击的示例商品入口，并写入一批真库演示数据，方便人工核对页面效果。
6. 补充复制操作，支持快速复制商品 ID、内容哈希、版本 ID 和 SKU ID。

### 用户的核心体验链路
1. 用户进入“商品内容资产”页面。
2. 输入 `platform_product_id`，默认平台为 `ozon`。
3. 页面同时读取该商品的最新内容资产和历史版本列表。
4. 用户先看到标题、描述、标签、主图、多图和 SKU 数量等概览信息。
5. 用户点击某个历史版本后，页面刷新主内容区和 SKU 区，核对该版本下的图片、价格和 SKU 映射关系。
6. 用户可同时查看该商品最近一次经营快照，辅助判断当前内容核对优先级。

## 模块二：【前端与组件设计】
### 路由与导航
- 新增路由：`/product-content`
- 在 `AppShell` 侧边导航新增“商品内容资产”入口

### 页面结构
- 页面 Hero：说明该页只用于核对内容资产版本和 SKU 映射，并直接暴露示例商品入口
- 检索面板：平台、商品 ID、查询按钮、重置按钮
- 示例商品面板：内置 3 个可点击的演示商品
- 内容概览面板：标题、描述、标签、主图、多图、内容哈希、采集时间
- 经营快照面板：销量、销售额、均价、毛利率、店铺、品牌、曝光点击和物流信息
- 版本列表面板：展示同商品的历史内容版本，支持切换
- SKU 面板：展示当前所选版本的 SKU 列表、价格、币种、图片数组

### 组件拆分
- 新增页面组件：`frontend/menglar-workbench/src/pages/ProductContentPage.jsx`
- 复用现有 `Panel` 组件承接面板结构
- 复用 `styles.css` 追加页面专用样式，不额外拆 CSS 文件
- 页面内补充复制按钮/复制卡片，便于人工验收时快速带走关键 ID

## 模块三：【后端与架构设计】
### API 设计
本轮页面只复用现有接口，不新增 API：
- `GET /api/product-content?productId=xxx&platform=ozon`
  - 读取某商品最新内容版本及其 SKU
- `GET /api/product-content?productId=xxx&platform=ozon&latest=false`
  - 读取某商品内容版本历史
- `GET /api/product-content/:id/skus`
  - 读取指定内容版本下的 SKU 列表
- `GET /api/product-business/latest?productId=xxx&platform=ozon`
  - 读取某商品最近一次经营快照

### 数据结构依赖
- `product_content_assets`
  - 承接产品级内容：标题、描述、标签、主图、多图、内容哈希、采集时间
- `product_content_skus`
  - 承接 SKU 级内容：平台 SKU ID、SKU 名称、价格、币种、图片数组

### 架构边界
- 页面仅做“读模型展示”，不做写入、编辑、版本合并
- 页面不引入新的数据表，不改变内容资产版本判定口径
- 页面展示“最新版本”仍以 `captured_at DESC, id DESC` 为准
- 演示数据通过独立种子脚本写入，不与正式采集逻辑耦合

# 萌拉数据工作台前端化改造

## 模块一：【需求概述与交互】
### 需求背景
当前“结果展示”和“采集任务”混在一个轻量静态页里，职责边界不清晰，不利于后续扩展成一体化工作台。用户已经明确要求把任务管理和结果浏览拆成两个页面，并把统计工具升级成专门的前端工作台。

### 一句话需求描述
使用专门的前端方案，将萌拉数据展示升级为工作台应用，并拆成“采集任务页”和“结果展示页”两个清晰路由。

### 解决方案大纲
1. 新建 React + Vite 前端工作台。
2. 使用 React Router 拆分两个页面：
   - `/tasks`
   - `/results`
3. 使用 TanStack Query 管理接口请求与缓存。
4. 保留现有 Node + SQLite 读取接口，前端只负责交互与展示。
5. 使用 Motion 做轻量过渡动画，为后续更完整的工作台视觉风格打底。

### 用户核心体验链路
1. 用户运行采集脚本生成 SQLite 数据。
2. 用户启动 API 服务和前端工作台。
3. 在“采集任务页”查看任务状态、错误信息、入库条数。
4. 在“结果展示页”查看统计指标、筛选结果和商品明细。

## 模块二：【前端与组件设计】
### 路由设计
- `/tasks`：采集任务页
- `/results`：结果展示页

### 组件设计
- `AppShell`：工作台外壳、侧边导航
- `TasksPage`：任务列表页
- `ResultsPage`：结果分析页
- `Panel`：统一面板容器
- `StatusBadge`：任务状态标签

### 技术选型
- 构建：Vite
- 框架：React
- 路由：React Router
- 数据请求：TanStack Query
- 动效：Motion

## 模块三：【后端与架构设计】
### 现有接口沿用
- `GET /api/jobs`
- `GET /api/products`

### 架构关系
- Node 服务负责：
  - 读取 SQLite
  - 提供 JSON 接口
- React 前端负责：
  - 页面路由
  - 数据可视化展示
  - 筛选交互

### 数据库设计
- 不新增表
- 继续读取：
  - `source_jobs`
  - `products_normalized`

### 埋点设计
- 本轮不新增埋点
- 保留页面控制台错误与服务端访问日志

# 项目说明与目录结构

## 1. 项目定位

本项目是一个面向跨境电商运营的本地工具型项目，当前核心围绕以下几类能力：

- 萌拉 / Menglar 数据采集、入库与结果查看。
- Ozon 官方物流费用规则导入、校验、试算与比价。
- Ozon 商品批量上货数据校验、模板生成与接口调用辅助。
- 本地电商资料、商品上架 SOP、选品分析文档沉淀。

项目整体采用 Node.js 作为脚本与后端运行环境，前端工作台使用 React + Vite，数据主要落在本地 SQLite 与 JSON / XLSX 文件中。当前更接近“本地运营工作台 + 数据脚本集合”，不是传统的线上多环境 Web 服务。

## 2. 技术栈与运行环境

- 运行环境：Node.js，项目使用 ESM 模块。
- 前端：React、React Router、TanStack React Query、Vite。
- 后端：Node.js 原生 HTTP 服务。
- 数据库：SQLite，本地数据库文件为 `db/menglar-mvp.sqlite`。
- 自动化验证：Node 脚本、Playwright。
- 主要配置：`package.json` 中维护统一命令入口。

首次接手建议先执行：

```powershell
npm install
npm run dev
```

默认访问地址：

- 前端工作台：`http://127.0.0.1:4178`
- 后端 API：`http://127.0.0.1:4186`

## 3. 快速启动命令

| 命令 | 用途 |
| --- | --- |
| `npm run dev` | 同时启动后端 API 与前端 Vite 开发服务 |
| `npm run workbench:api` | 只启动工作台后端 API，默认端口 `4186` |
| `npm run workbench:web` | 只启动前端 Vite 服务，默认端口 `4178` |
| `npm run workbench:web:build` | 构建前端工作台到 `frontend/menglar-workbench/dist` |
| `npm run mvp:probe` | 执行萌拉采集最小链路 |
| `npm run mvp:db:inspect` | 查看萌拉 SQLite 数据摘要 |
| `npm run shipping:import` | 从 Ozon 官方 XLSX 导入物流规则，生成 `rules.generated.json` |
| `npm run shipping:validate` | 校验当前生效的物流规则文件 |
| `npm run shipping:validate:generated` | 校验脚本生成的物流规则文件 |
| `npm run shipping:api:test` | 执行物流规则 API 自测 |
| `npm run shipping:page:test` | 执行物流计算页面自测 |
| `npm run ozon:tool` | 执行 Ozon 批量上货工具脚本 |
| `npm run ozon:tool:test` | 执行 Ozon 上货工具脚本测试 |
| `npm run ozon:api:test` | 执行 Ozon 工作台 API 自测 |
| `npm run ozon:page:test` | 执行 Ozon 上传页面自测 |

更细的命令说明可参考 `docs/常用命令.md`。若发现该文件显示乱码，优先以 `package.json` 中脚本为准。

## 5. 核心目录说明

### 5.1 `backend/`

后端服务目录，当前只有一个工作台 API 服务：

```text
backend/
└─ menglar-workbench-api/
   ├─ server.mjs                 # 后端 HTTP 服务入口，负责 API 路由与静态资源托管
   └─ lib/
      └─ shipping-engine.mjs     # Ozon 物流费用计算、比价、规则查询核心逻辑
```

`server.mjs` 默认读取：

- SQLite 数据库：`db/menglar-mvp.sqlite`
- 前端构建产物：`frontend/menglar-workbench/dist`
- 物流规则配置：由 `shipping-engine.mjs` 间接读取 `config/shipping/`

当前后端主要 API：

- `GET /api/jobs`
- `GET /api/products`
- `GET /api/ozon/template`
- `POST /api/ozon/validate`
- `POST /api/ozon/execute`
- `POST /api/ozon/import-info`
- `POST /api/ozon/category-attributes`
- `POST /api/ozon/attribute-values`
- `GET /api/shipping/methods`
- `GET /api/shipping/rule-info`
- `POST /api/shipping/calculate`
- `POST /api/shipping/calculate-batch`
- `POST /api/shipping/compare`

接口字段与维护要求参考 `docs/API接口文档.md`。涉及 API 增删改时，必须同步更新该文档。

### 5.2 `frontend/`

前端工作台目录：

```text
frontend/
└─ menglar-workbench/
   ├─ index.html
   ├─ vite.config.mjs
   └─ src/
      ├─ main.jsx                    # React 入口与路由定义
      ├─ styles.css                  # 全局样式
      ├─ components/                 # 通用 UI 组件
      ├─ lib/                        # 前端 API 请求与格式化工具
      ├─ pages/                      # 页面级组件
      └─ shell/                      # 工作台整体布局
```

当前页面路由：

| 路由 | 页面文件 | 用途 |
| --- | --- | --- |
| `/tasks` | `TasksPage.jsx` | 萌拉采集任务列表 |
| `/results` | `ResultsPage.jsx` | 萌拉采集商品结果查看 |
| `/shipping-calculator` | `ShippingCalculatorPage.jsx` | Ozon 物流费用试算与比价 |
| `/shipping-calculator/rules` | `ShippingRuleInfoPage.jsx` | 物流规则信息查看 |
| `/ozon-upload` | `OzonUploadPage.jsx` | Ozon 批量上货工具页面 |

前端 API 请求统一放在 `frontend/menglar-workbench/src/lib/api.js`。开发环境下 Vite 将 `/api` 代理到 `http://127.0.0.1:4186`。

### 5.3 `scripts/`

脚本目录是本项目的关键维护入口，包含采集、导入、校验、测试等任务：

```text
scripts/
├─ dev.mjs                         # 同时启动后端与前端开发服务
├─ menglar-mvp.mjs                 # 萌拉采集最小链路脚本
├─ inspect-menglar-db.mjs          # 查看 SQLite 数据摘要
├─ menglar-industry-deep-dive.mjs  # 萌拉行业机会分析脚本
├─ import-shipping-rules.mjs       # 导入 Ozon 官方物流 XLSX 并生成规则
├─ validate-shipping-config.mjs    # 校验物流规则 JSON
├─ test-shipping-api.mjs           # 物流 API 自测
├─ test-shipping-page.mjs          # 物流页面自测
├─ ozon-batch-tool.mjs             # Ozon 批量上货工具脚本
├─ test-ozon-batch-tool.mjs        # Ozon 上货工具脚本测试
├─ test-ozon-workbench-api.mjs     # Ozon 工作台 API 测试
├─ test-ozon-upload-page.mjs       # Ozon 上传页面测试
└─ lib/
   └─ ozon-seller-client.mjs       # Ozon Seller API 客户端与数据校验逻辑
```

项目规范要求：后续新增脚本应放到 `scripts/[功能中文名]/` 下，并补充 README 说明，方便复用。当前已有部分历史脚本仍在 `scripts/` 根目录，维护时应逐步收敛。

### 5.4 `config/`

业务配置目录，当前主要是 Ozon 物流规则：

```text
config/
└─ shipping/
   ├─ README.md
   ├─ rules.json                    # 当前生效的本地物流规则
   ├─ rules.generated.json          # 从官方 XLSX 生成的候选规则
   ├─ fx.json                       # 汇率配置
   └─ calculator-calibration.json   # 官方计算器抽样校准数据
```

维护原则：

- `rules.json` 是当前生效规则。
- `rules.generated.json` 是脚本生成结果，不应未经抽样校验直接替换 `rules.json`。
- 官方 XLSX 源文件放在 `data/shipping-sources/`。
- 更新规则后需要执行 `shipping:validate`、`shipping:validate:generated` 以及相关 API / 页面测试。

### 5.5 `data/`

本地数据目录，包含采集结果、日志、官方源文件与示例数据：

```text
data/
├─ logs/                  # 本地开发服务日志
├─ menglar-industry/      # 萌拉行业分析结果
├─ menglar-mvp/           # 萌拉采集运行产物与数据库副本
├─ ozon-upload/           # Ozon 上货工具示例数据
└─ shipping-sources/      # Ozon 官方物流 XLSX 源文件
```

注意：

- `data/logs/` 是运行日志，通常不作为功能交付内容。
- `data/shipping-sources/` 是物流规则生成的重要源数据。
- `data/menglar-mvp/last-run.json` 可用于快速查看最近一次采集结果。

### 5.6 `db/`

数据库目录：

```text
db/
```

维护约定：

该目录与代码研发文档分开，适合存放：

## 6. 数据流与功能关系

### 6.1 萌拉采集与结果展示

```text
scripts/menglar-mvp.mjs
  -> 抓取萌拉页面数据
  -> 写入 db/menglar-mvp.sqlite
  -> backend/menglar-workbench-api/server.mjs 提供 /api/jobs 与 /api/products
  -> frontend/menglar-workbench 展示任务和商品结果
```

相关页面：

- `/tasks`
- `/results`

相关数据库表：

- `source_jobs`
- `products_raw`
- `products_normalized`

### 6.2 Ozon 物流费用规则

```text
data/shipping-sources/*.xlsx
  -> scripts/import-shipping-rules.mjs
  -> config/shipping/rules.generated.json
  -> 人工抽样校验
  -> config/shipping/rules.json
  -> backend/lib/shipping-engine.mjs
  -> /api/shipping/*
  -> /shipping-calculator 与 /shipping-calculator/rules
```

相关测试：

- `npm run shipping:validate`
- `npm run shipping:validate:generated`
- `npm run shipping:api:test`
- `npm run shipping:page:test`

### 6.3 Ozon 批量上货工具

```text
data/ozon-upload/*.json
  -> scripts/lib/ozon-seller-client.mjs
  -> scripts/ozon-batch-tool.mjs
  -> backend /api/ozon/*
  -> frontend /ozon-upload
```

相关测试：

- `npm run ozon:tool:test`
- `npm run ozon:api:test`
- `npm run ozon:page:test`

## 7. 接手建议流程

1. 先阅读本文件、`AGENTS.md`、`package.json`、`docs/API接口文档.md`。
2. 执行 `npm install`，确认依赖可安装。
3. 执行 `npm run dev`，打开 `http://127.0.0.1:4178` 查看工作台。
4. 查看 `docs/需求文档/`，理解历史功能的方案背景。
5. 查看 `docs/测试文档/`，确认已有功能的验证方式。
6. 若接手物流规则，先读 `config/shipping/README.md` 与 `data/shipping-sources/README.md`。
7. 若接手数据库相关功能，先读 `docs/db/数据库说明文档.md` 和各表文档。
8. 修改前运行 `git status`，避免把他人未提交改动混入本次任务。

## 8. 开发与维护注意事项

- 本项目当前存在运行产物、运营素材、测试截图和业务文档混合存放的情况，提交前必须明确哪些文件属于本次变更。
- 不要直接提交 `node_modules/`、临时缓存、无关日志。
- API 改动必须同步维护 `docs/API接口文档.md`。
- 数据库结构或字段含义改动必须同步维护 `docs/db/`。
- 前端页面结构、复杂交互、工作台类改动，应同步维护 `docs/需求文档/` 下的页面结构与线框方案。
- 涉及数据库写入的功能，自测不能只看页面 200 或接口 200，必须直连查库核验字段。
- 涉及 UI 布局、报告页、弹窗、表单主交互变化时，应按项目规则补充 UI 截图回归产物到对应 `docs/测试文档/[功能名称]/UI/`。

## 9. 常见问题定位

| 问题 | 优先检查 |
| --- | --- |
| 前端页面打不开 | `npm run workbench:web` 是否启动，端口 `4178` 是否被占用 |
| 前端请求 API 失败 | `npm run workbench:api` 是否启动，端口 `4186` 是否正常 |
| `/api/products` 没有数据 | `db/menglar-mvp.sqlite` 是否存在，是否有成功采集任务 |
| 物流试算结果异常 | `config/shipping/rules.json`、`fx.json`、`calculator-calibration.json` |
| 物流导入结果异常 | `data/shipping-sources/` 官方 XLSX 是否更新，执行 `shipping:validate:generated` |
| Ozon 接口调用失败 | API 凭据、请求模式、`scripts/lib/ozon-seller-client.mjs` 校验逻辑 |
| 页面测试失败 | 后端和前端服务是否都启动，Playwright 浏览器依赖是否可用 |

## 10. 推荐阅读顺序

新成员不需要一开始读完所有代码，建议按任务方向选择：

- 只看整体：本文件 -> `package.json` -> `docs/常用命令.md`
- 做前端：`frontend/menglar-workbench/src/main.jsx` -> `src/shell/AppShell.jsx` -> 对应 `src/pages/*.jsx` -> `src/lib/api.js`
- 做后端 API：`backend/menglar-workbench-api/server.mjs` -> `docs/API接口文档.md`
- 做物流规则：`config/shipping/README.md` -> `data/shipping-sources/README.md` -> `backend/menglar-workbench-api/lib/shipping-engine.mjs`
- 做 Ozon 上货：`scripts/lib/ozon-seller-client.mjs` -> `scripts/ozon-batch-tool.mjs` -> `frontend/menglar-workbench/src/pages/OzonUploadPage.jsx`
- 做萌拉采集：`scripts/menglar-mvp.mjs` -> `scripts/inspect-menglar-db.mjs` -> `docs/db/`
